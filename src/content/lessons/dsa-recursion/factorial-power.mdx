## The hook: how many ways can you shuffle a deck?

A deck of 52 cards can be arranged in $52!$ ways — a number with 68 digits, more than the count of
atoms you'd find in a galaxy. That `!` is **factorial**, and it's the textbook recursion: $n!$ is just
$n$ times the factorial of the slightly smaller number $n-1$. Then we'll meet its cleverer cousin:
raising a number to a power. The obvious way multiplies $n$ times, but a recursive trick called
**exponentiation by squaring** gets there in roughly $\log_2 n$ steps — the same idea that lets your
browser do RSA encryption fast enough to be invisible.

<Callout type="intuition" title="The one-sentence idea">
**Factorial** multiplies a shrinking chain ($n \cdot (n{-}1) \cdots 1$); **fast power** halves the
exponent each step, turning $n$ multiplications into about $\log_2 n$.
</Callout>

## Intuition: shrink by one vs shrink by half

**Factorial** is the most direct recursion imaginable. The base case is $0! = 1! = 1$. The recursive
case is $n! = n \cdot (n-1)!$. Each call peels off one factor, so reaching the base case from $n$ takes
$n$ steps — a long, straight chain.

**Power** can do far better. To compute $x^n$, notice that

$$
x^n = \begin{cases} 1 & n = 0 \\ \left(x^{n/2}\right)^2 & n \text{ even} \\ x \cdot x^{\,n-1} & n \text{ odd} \end{cases}
$$

The magic is the **even** case: instead of recursing from $n$ to $n-1$ (shrink by one), it recurses
from $n$ to $n/2$ (shrink by **half**). Compute the half once, then **square** it — one multiplication
buys you a doubling of the exponent. Halving repeatedly reaches $0$ in about $\log_2 n$ steps, so the
stack stays tiny even for huge exponents.

## How it works: a dry run

`power(2, 13)` with the squaring trick. Watch the exponent collapse:

- $13$ is odd → $2 \cdot \text{power}(2, 12)$
- $12$ is even → $\text{power}(2, 6)^2$
- $6$ is even → $\text{power}(2, 3)^2$
- $3$ is odd → $2 \cdot \text{power}(2, 2)$
- $2$ is even → $\text{power}(2, 1)^2$
- $1$ is odd → $2 \cdot \text{power}(2, 0)$, and $\text{power}(2, 0) = 1$

Six levels of depth for exponent 13 — versus thirteen for the naive multiply-one-at-a-time approach.

## The real implementation

Faithful excerpts from the course files. Notice `powerRecursive` is exactly the even/odd split above.

```ts
// /dsa/04-recursion/factorial.ts
// Basic recursive factorial — Time: O(n), Space: O(n)
function factorialRecursive(n: number): number {
  if (n < 0) throw new Error('Factorial not defined for negative numbers');
  if (n <= 1) return 1;
  return n * factorialRecursive(n - 1);
}

// /dsa/04-recursion/power-function.ts
// Binary exponentiation, recursive — Time: O(log n), Space: O(log n)
function powerRecursive(x: number, n: number): number {
  if (n < 0) { x = 1 / x; n = -n; }   // x^-n = (1/x)^n
  if (n === 0) return 1;
  if (n % 2 === 0) {
    const half = powerRecursive(x, n / 2);
    return half * half;               // one squaring doubles the exponent
  }
  return x * powerRecursive(x, n - 1);
}
```

Run the full files: `npx tsx 04-recursion/factorial.ts` and `npx tsx 04-recursion/power-function.ts`

## Complexity

| Algorithm | Best | Average | Worst | Stack space |
| --- | --- | --- | --- | --- |
| `factorial(n)` | $O(n)$ | $O(n)$ | $O(n)$ | $O(n)$ |
| Naive power $x^n$ | $O(n)$ | $O(n)$ | $O(n)$ | $O(n)$ |
| Fast power (squaring) | $O(\log n)$ | $O(\log n)$ | $O(\log n)$ | $O(\log n)$ |

Factorial obeys $T(n) = T(n-1) + O(1) = O(n)$: shrink by one, $n$ steps. Fast power obeys

$$
T(n) = T(n/2) + O(1) \;\Rightarrow\; T(n) = O(\log n),
$$

because halving $n$ until it reaches $0$ takes $\lfloor \log_2 n \rfloor + 1$ steps.

<Callout type="why">
This $T(n/2)$ pattern — recurse on half the input — is the heartbeat of <strong>divide and conquer</strong>:
binary search, merge sort, and balanced-tree operations all share it. Recognising "I can shrink by half
instead of by one" is one of the highest-leverage instincts in algorithms.
</Callout>

## The same idea, different worlds

<ExampleCard domain="Cryptography" icon="🔐">
RSA encrypts by computing $m^e \bmod N$ where the exponent $e$ has hundreds of digits. The naive
"multiply $e$ times" approach would never finish before the heat death of the universe. Fast power —
$O(\log e)$ squarings, each kept modulo $N$ — makes it instant. Every HTTPS handshake leans on this.
</ExampleCard>

<ExampleCard domain="Probability" icon="🎲">
Factorials count arrangements. The chance of dealing a *specific* ordering of 5 cards from a deck
involves $52! / 47!$. Casinos, lotteries, and poker odds all rest on $n!$ counting "how many ways can
this happen?"
</ExampleCard>

<ExampleCard domain="Computer graphics" icon="🎞️">
Smoothly blending between keyframes uses Bézier curves built from **Bernstein polynomials**, whose
coefficients are binomial numbers — and those are ratios of factorials. The `n!` you compute here is
quietly shaping the easing of every animated transition.
</ExampleCard>

## Common pitfalls & interview tips

- **Integer overflow.** $n!$ explodes past `Number.MAX_SAFE_INTEGER` around $n = 18$; the course code
  offers `factorialBigInt` for exact large results. Interviewers may ask you to return the result
  *modulo* a prime to keep it bounded — combine that with fast power.
- **The even case must square, not double.** A classic bug is writing `power(x, n / 2) * 2` instead of
  `half * half`. You're squaring the *value*, which doubles the *exponent*.
- **Forgetting `half` is computed once.** If you write `power(x, n/2) * power(x, n/2)` you make **two**
  recursive calls and lose the speedup — back to $O(n)$. Store it in a variable first.
- **Negative or fractional exponents.** Handle $n < 0$ via $x^{-n} = (1/x)^n$, as the real code does.
- **Interview tip:** if asked to compute a power, mention exponentiation by squaring and its
  $O(\log n)$ cost even before coding — it's the signal that separates "I can loop" from "I know the
  structure."

## Practice

1. **Tail-recursive factorial (easy).** Rewrite factorial to carry an accumulator,
   `factorial(n, acc)`, so the multiplication happens *before* the recursive call. (We'll see why this
   matters in the recursion-vs-iteration lesson.)
2. **Count the multiplications (easy).** Instrument fast power to return how many multiplications it
   used, and confirm it tracks $\log_2 n$.
3. **Modular fast power (medium).** Compute $x^n \bmod m$ by taking the remainder after each
   multiplication — the cryptography workhorse.
4. **Sum of digits via division (easy, from `recursion-practice.js`).** Same "shrink the number by a
   factor each step" instinct as fast power, applied to base-10 digits.
