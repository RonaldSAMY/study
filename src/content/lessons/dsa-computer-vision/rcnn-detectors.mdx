## The hook: from 47 seconds to real time

The first deep detector, R-CNN (2014), was breathtakingly accurate and painfully slow: it cropped
around **two thousand** region proposals per image and ran a full CNN on each one — about 47 seconds per
photo. Over three years the R-CNN family fixed this by asking a sharper question each time. Share the
convolution across all regions (Fast R-CNN). Then *learn* the proposals with a tiny network instead of
a slow external algorithm (Faster R-CNN). The result is the two-stage detector that still anchors
accuracy-first systems and Mask R-CNN today.

<Callout type="intuition" title="The one-sentence idea">
A two-stage detector first proposes where objects might be by scoring a grid of anchors and nudging them
onto nearby objects, then a second head classifies each surviving proposal and refines its box.
</Callout>

## Intuition

Stage one is the **Region Proposal Network** (RPN). Recall the anchors tiled across a feature map: the
RPN slides over that map and, at every location, asks two questions per anchor — "is there an object
here?" (objectness) and "how should I nudge this anchor to fit it?" (four box deltas). Anchors are
labelled by their IoU with ground truth: high overlap is a **positive** target, low overlap is
**background**, and the ambiguous middle is **ignored**. The positives, refined and de-duplicated with
[NMS](/lesson/dsa-computer-vision/cv-foundations), become **region proposals**.

The crucial trick is that a box is never predicted from raw coordinates. Instead the network predicts a
small **correction** to an anchor: shift the center (normalized by anchor size) and scale the width and
height in log space. Log space means the network's natural zero output says "keep the anchor as is", and
a box twice as large and half as large are symmetric corrections. Stage two then pools features from
each proposal — with **RoIAlign**, which samples at exact float coordinates instead of rounding — and
runs a classifier plus a per-class box refiner.

## How it works

Anchor A has center (50, 50) and size 40×40; ground truth G has center (65, 60) and size 50×40. Encode
the regression targets, then decode a network prediction back to a box:

```
t_x = (65-50)/40 = 0.375     t_y = (60-50)/40 = 0.250
t_w = log(50/40) = 0.22314   t_h = log(40/40) = 0.0

predict t = (0.40, 0.20, 0.25, 0.05)  ->
pred_x = 50 + 0.40*40 = 66      pred_w = 40*exp(0.25) = 51.36
pred_y = 50 + 0.20*40 = 58      pred_h = 40*exp(0.05) = 42.05
```

## The real implementation

The course encodes and decodes with exactly the log/normalize parameterization above:

```ts
// /dsa/32-computer-vision/03-context-detectors/rcnn-family.ts
export function encodeBox(anchor: Box, gt: Box) {
  const A = cornersToCenter(anchor), G = cornersToCenter(gt);
  return [ (G.cx - A.cx) / A.w, (G.cy - A.cy) / A.h, Math.log(G.w / A.w), Math.log(G.h / A.h) ];
}
export function decodeBox(anchor: Box, t: number[]): Box {
  const A = cornersToCenter(anchor);
  return centerToCorners({
    cx: A.cx + t[0] * A.w, cy: A.cy + t[1] * A.h,
    w: A.w * Math.exp(t[2]), h: A.h * Math.exp(t[3]),
  });
}
```

The RPN multi-task loss combines binary cross-entropy objectness with smooth-L1 box regression, and the
box term fires **only for positive anchors** — there is no correct box for background:

```ts
// /dsa/32-computer-vision/03-context-detectors/rcnn-family.ts
for (const a of anchors) {
  if (a.label === -1) continue;                 // ignored anchors: no loss
  clsSum += bceLoss(a.p, a.label); nCls++;
  if (a.label === 1) regSum += boxRegLoss(a.t, a.tStar);   // positives only
  nReg++;
}
// L = (1/N_cls) * clsSum  +  lambda * (1/N_reg) * regSum
```

## Formulas and complexity

Box regression targets normalize the center shift by anchor size and take the log of the size ratio:

$$
t_x = \frac{G_x - A_x}{A_w}, \quad t_y = \frac{G_y - A_y}{A_h}, \quad
t_w = \log\frac{G_w}{A_w}, \quad t_h = \log\frac{G_h}{A_h}.
$$

The regression uses the smooth-L1 (Huber) function — quadratic near zero for stable gradients, linear
far out for robustness:

$$
\text{smooth}_{L_1}(x) = \begin{cases} 0.5\,x^2 & |x| < 1 \\ |x| - 0.5 & \text{otherwise.} \end{cases}
$$

The full RPN objective is a weighted sum of a classification and a regression term:

$$
L = \frac{1}{N_{\text{cls}}} \sum_i L_{\text{cls}}(p_i, p_i^\star)
  + \lambda \, \frac{1}{N_{\text{reg}}} \sum_i p_i^\star \, L_{\text{reg}}(t_i, t_i^\star).
$$

| Stage | Time | Note |
|-------|------|------|
| R-CNN (test) | $O(R \cdot \text{CNN})$, $R \approx 2000$ | a full CNN per region |
| Fast R-CNN | $O(\text{CNN} + R \cdot \text{head})$ | one shared CNN pass |
| Faster R-CNN | $O(\text{CNN} + \text{RPN} + R \cdot \text{head})$ | proposals nearly free |
| RoI Align (one) | $O(H_o W_o \cdot \text{samples} \cdot C)$ | per proposal |

## Common pitfalls & interview tips

- **Predict deltas, never raw coordinates.** Regressing absolute pixels is unstable across scales; the
  normalized, log-space parameterization is what makes training converge.
- **The box term is masked to positives.** Multiplying by $p_i^\star$ means background anchors
  contribute no regression loss — there is no "correct box" for empty space.
- **RoI Pooling misaligns; RoIAlign fixes it.** Pooling rounds RoI edges to integer cells, shifting
  features by up to a stride. Mask R-CNN's masks only became sharp once RoIAlign removed that rounding.
- **Interview cue:** the one-line story of the family is "R-CNN runs a CNN per region; Fast shares the
  CNN with RoI pooling; Faster learns the proposals with an RPN so proposals become nearly free."
