## The hook: how much of me looks like my own beginning?

Slide along a string and, at every spot, ask one question: *starting here, how many characters match the very start of the string?* That single number, computed for every position, is the **Z-array**. It sounds modest, but it quietly answers "where does this pattern occur?", "does this string repeat?", and "what is its shortest period?" — all in time proportional to the length of the input, with no hashing and no backtracking.

The naive way re-compares from scratch at every position and costs $O(n^2)$. The Z-algorithm gets it in $O(n)$ by never re-examining a character it has already confirmed — it remembers one clever interval and reuses it.

<Callout type="intuition" title="The one-sentence idea">
Keep the rightmost stretch you already know matches the prefix (the "Z-box"); when a new position falls inside it, copy the answer from its mirror instead of comparing characters again.
</Callout>

## Intuition

You already met **KMP**, which pre-processes a pattern into a failure function that says "after a mismatch, where do I resume?". The Z-algorithm attacks the same problem from a friendlier angle. Instead of "where do I resume after failing", it directly reports "how much matches the prefix starting here". That quantity, $Z[i]$, is often easier to reason about.

Here is the trick that makes it linear. Suppose we have already discovered a substring $s[L\dots R]$ that equals a prefix of $s$ — call it the **Z-box**, and keep the one with the largest $R$. Now we want $Z[i]$ for some $i$ inside that box. Because $s[L\dots R]$ *is* a prefix, the character at $i$ has a **mirror** at position $i - L$ near the front of the string, and we already computed $Z[i-L]$. If that mirrored value fits comfortably inside the box, we simply copy it — zero comparisons. Only when the mirror runs into the box's edge do we resume comparing characters, and every such comparison pushes $R$ forward. Since $R$ only ever increases, the total work is $O(n)$.

## How it works

Take `s = "aabaaab"`. We scan `i = 1..6`, keeping the Z-box `[L, R]`:

```
i=1: 'a' matches prefix 'a', then 'a' vs 'b' stop → Z[1]=1, box [1,1]
i=2: 'b' vs 'a' stop                             → Z[2]=0
i=3: 'aa' matches 'aa', 'a' vs 'b'? → Z[3]=2, box [3,4]
i=4: inside box, mirror Z[1]=1 fits             → Z[4]=1  (copied, no compares)
i=5: 'a' vs 'a', 'a' vs 'a', 'b' vs 'b' … → Z[5]=2, box [5,6]
i=6: 'b' vs 'a' stop                             → Z[6]=0
```

For **pattern matching**, glue the pattern, a separator that appears nowhere, and the text into one string: `pattern + "$" + text`. Any position where $Z[i]$ equals the pattern length is a full occurrence. Run it:

function buildZ(s) {
  const n = s.length;
  const z = new Array(n).fill(0);
  let L = 0, R = 0;
  for (let i = 1; i < n; i++) {
    if (i <= R) z[i] = Math.min(R - i + 1, z[i - L]);
    while (i + z[i] < n && s[z[i]] === s[i + z[i]]) z[i]++;
    if (i + z[i] - 1 > R) { L = i; R = i + z[i] - 1; }
  }
  return z;
}

// Find every occurrence of pattern in text via Z on pattern + "$" + text.
function zSearch(text, pattern) {
  const combined = pattern + "$" + text;
  const z = buildZ(combined);
  const m = pattern.length, hits = [];
  for (let i = m + 1; i < combined.length; i++)
    if (z[i] === m) hits.push(i - m - 1);
  return hits;
}`}

## The real implementation

The course file keeps $L$ and $R$ explicit and branches on "inside vs outside the box", which mirrors the intuition closely:

```ts
// /dsa/18-string-algorithms/z-algorithm.ts
export function buildZArray(s: string): number[] {
  const n = s.length;
  const z: number[] = new Array(n).fill(0);
  if (n === 0) return z;

  let L = 0, R = 0;                       // Z-box boundaries
  for (let i = 1; i < n; i++) {
    if (i > R) {                          // outside the box: from scratch
      L = R = i;
      while (R < n && s[R - L] === s[R]) R++;
      z[i] = R - L; R--;
    } else {                              // inside the box
      const k = i - L;                    // mirror position in the prefix
      if (z[k] < R - i + 1) z[i] = z[k];  // mirror fits → copy, no compares
      else {                              // mirror hits the edge → extend
        L = i;
        while (R < n && s[R - L] === s[R]) R++;
        z[i] = R - L; R--;
      }
    }
  }
  return z;
}
```

## Complexity

The right boundary $R$ is monotonic — it only moves rightward across the whole run — and each character comparison either fails once or advances $R$. So the comparisons sum to $O(n)$:

$$
T_{\text{build}}(n) = O(n), \qquad T_{\text{search}}(n, m) = O(n + m), \qquad S(n) = O(n).
$$

| Task | Time | Space |
|------|------|-------|
| Naive prefix matching | $O(n^2)$ | $O(1)$ |
| Build Z-array | $O(n)$ | $O(n)$ |
| Pattern search (pattern + `$` + text) | $O(n + m)$ | $O(n + m)$ |

<ExampleCard domain="Music production" icon="🎧">
A looped sample is a string whose shortest period you want to know. Because a period $p$ exists exactly when $Z[i] + i = n$ for some $i$, one Z pass tells a DAW the tightest loop length — so it can trim a bar to its true repeating unit without guesswork.
</ExampleCard>

<ExampleCard domain="Text editors" icon="⌨️">
Autocomplete and "find next" need every place the typed prefix reappears. Concatenating the query with the document and scanning for full-length Z values surfaces those hits in a single linear sweep, keeping the cursor responsive on large files.
</ExampleCard>

## Common pitfalls & interview tips

- **Pick a separator that truly cannot occur.** If `$` appears in the text, a spurious $Z[i] = m$ can slip through. Use a sentinel outside both alphabets (or check bounds explicitly).
- **$Z[0]$ is a special case.** By convention it is $0$ (or "the whole string"); never let the loop start at $0$, or the mirror logic breaks.
- **Copy versus extend is the whole game.** Inside the box, copy only when `z[k] < R - i + 1`; otherwise you *must* keep comparing from `R + 1`. Getting that inequality wrong silently returns short answers.
- **Interview cue:** periods, "is this a rotation/repeat", and "longest prefix that is also a suffix" all reduce to the identity $Z[i] + i = n$. Say that out loud and you have the solution skeleton.
