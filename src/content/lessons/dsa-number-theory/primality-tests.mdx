## The hook: your browser just tested a 600-digit number

Every time you open an `https://` page, somewhere a computer generated a fresh random number hundreds of
digits long and asked "is this prime?" — and answered in milliseconds. You obviously cannot trial-divide
a 600-digit number; the universe would end first. The trick is a *probabilistic* test that interrogates
the number with a few random "witnesses." If any witness catches it lying, it is composite for sure; if
enough witnesses fail to catch it, it is prime with overwhelming confidence.

<Callout type="intuition" title="The one-sentence idea">
Instead of hunting for a factor, Miller-Rabin picks random bases $a$ and checks an identity that *all*
primes must satisfy; a single base that violates it proves the number composite.

</Callout>

## Intuition

Fermat's little theorem says that for a prime $p$ and any $a$ not divisible by $p$,
$a^{p-1} \equiv 1 \pmod p$. Turn it around: if $a^{n-1} \not\equiv 1 \pmod n$ for some $a$, then $n$ is
*definitely* not prime. That is the Fermat test — fast, but it has a fatal flaw: **Carmichael numbers**
like 561 and 1729 satisfy the identity for every base yet are composite.

Miller-Rabin patches the hole with one extra idea. Factor out the twos: write $n - 1 = 2^s \cdot d$ with
$d$ odd. For a real prime, the chain

$$
a^{d}, \; a^{2d}, \; a^{4d}, \; \dots, \; a^{2^{s-1} d} \pmod n
$$

must either start at 1, or hit $n-1$ somewhere before it can reach 1. Why? Because modulo a prime the only
square roots of 1 are $\pm 1$. If you ever see a 1 appear *without* a preceding $n-1$, you have found a
"nontrivial square root of 1" — impossible for a prime — so $n$ is composite, and $a$ is the *witness*
that proves it.

## How it works

Test $n = 221 = 13 \times 17$. Then $n - 1 = 220 = 2^2 \cdot 55$, so $s = 2$, $d = 55$. Try witness $a = 174$:

```
x0 = 174^55 mod 221 = 47      (not 1, not 220)
x1 = 47^2   mod 221 = 220     (= n-1)  ->  passes for a = 174
```

So 174 fails to expose 221. But most bases do expose it — try $a = 137$ and the chain never reaches 220,
proving 221 composite. For a real prime like 220 nothing ever catches it, no matter the base. Running $k$
random witnesses drops the chance of a wrong "prime" to at most $4^{-k}$.

## The real implementation

The course file uses `bigint` for the witness loop and a fixed witness set to make the test
*deterministic* below a huge bound:

```ts
// /dsa/22-number-theory/primality-tests.ts
function millerRabinWitness(a: bigint, d: bigint, s: bigint, n: bigint): boolean {
  let x = modPowBigInt(a, d, n);
  if (x === 1n || x === n - 1n) return true;
  for (let r = 1n; r < s; r++) {
    x = (x * x) % n;
    if (x === n - 1n) return true;    // possibly prime
    if (x === 1n) return false;       // definitely composite
  }
  return false;                       // definitely composite
}

export function isPrimeDeterministic(n: number): boolean {
  if (n < 2) return false;
  if (n % 2 === 0) return n === 2;
  let s = 0, d = n - 1;
  while (d % 2 === 0) { d /= 2; s++; }
  // These specific bases prove primality for all n below ~3.3·10^24
  const witnesses = [2n, 3n, 5n, 7n, 11n, 13n, 17n, 19n, 23n, 29n, 31n, 37n];
  for (const a of witnesses) {
    if (a >= BigInt(n)) continue;
    if (!millerRabinWitness(a, BigInt(d), BigInt(s), BigInt(n))) return false;
  }
  return true;
}
```

## Complexity

Each witness is one modular exponentiation plus up to $s$ squarings, and $s < \log_2 n$:

$$
T_{\text{trial}}(n) = O(\sqrt n), \qquad T_{\text{Miller-Rabin}}(n) = O\!\left(k \log^3 n\right).
$$

| Algorithm | Type | Time | Notes |
|-----------|------|------|-------|
| Trial division | Deterministic | $O(\sqrt n)$ | Fine for small $n$ |
| Fermat | Probabilistic | $O(k \log^2 n)$ | Fooled by Carmichael numbers |
| Miller-Rabin ($k$ random) | Probabilistic | $O(k \log^3 n)$ | Error $\le 4^{-k}$ |
| Miller-Rabin (fixed bases) | Deterministic | $O(\log^3 n)$ | Exact below a known bound |

For a 600-digit RSA prime, $\sqrt n$ is astronomically infeasible while $\log^3 n$ is a few thousand
operations — the entire reason public-key crypto is practical.

## The same idea, different worlds

<ExampleCard domain="Cryptography" icon="🔐">
RSA and elliptic-curve key generation repeatedly draw a random odd number and Miller-Rabin it until a
prime appears. Get the test wrong and you ship a factorable key.
</ExampleCard>

<ExampleCard domain="Hashing" icon="🧱">
Hash tables and universal hashing pick a *prime* table size or modulus to spread keys evenly. Libraries
keep a list of primes, or test candidates on the fly with exactly this routine.
</ExampleCard>

<ExampleCard domain="Distributed systems" icon="🌐">
Randomized primality is a poster child for Monte Carlo algorithms: fast, occasionally wrong in a
*one-sided* way, and made arbitrarily reliable by repetition — the same trade-off behind Bloom filters
and randomized consensus.
</ExampleCard>

## Common pitfalls & interview tips

- **Fermat is not enough.** Carmichael numbers (561, 1105, 1729, ...) pass Fermat for every coprime base.
  Always use Miller-Rabin when correctness matters.
- **Overflow in the squaring.** $x \cdot x$ for large $x$ overflows 64-bit integers; use `bigint` or
  128-bit multiplication, exactly as the reference does.
- **Order of the checks matters.** You must see $n-1$ *before* a 1 appears. Reaching 1 without a prior
  $n-1$ is the composite signal — swapping the checks breaks the test.
- **Interview cue:** "generate a large prime" or "is this number prime for $n$ up to $10^{18}$?" wants
  deterministic Miller-Rabin with a fixed witness set, not $O(\sqrt n)$ trial division.

## Practice

1. **Easy:** Use trial division with the $6k \pm 1$ pattern to test 91 and 97 by hand.
2. **Easy:** Factor $560 = n - 1$ for $n = 561$ into $2^s \cdot d$ and give $s$ and $d$.
3. **Medium:** Show that base $a = 2$ exposes 561 as composite by tracing the squaring chain.
4. **Medium:** Explain why a "nontrivial square root of 1" cannot exist modulo a prime.
5. **Hard:** Implement deterministic Miller-Rabin for all $n < 3{,}215{,}031{,}751$ using the witness set
   $\{2, 3, 5, 7\}$ and test it against a sieve.
