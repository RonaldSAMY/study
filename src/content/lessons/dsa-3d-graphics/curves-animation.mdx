## The hook: four dots, infinite smooth motion

A camera glides along a cinematic arc. A character's hand swings up to wave. A font's letter `S` curls
just so. None of these store thousands of positions — each keeps a *handful* of **control points** and a
rule for blending between them. Slide a single knob, the parameter `t`, from 0 to 1 and a smooth path
unspools point by point. The whole trick is built from one humble operation you already know: the
**lerp** — a weighted average of two points.

<Callout type="intuition" title="The one-sentence idea">
A curve is just lerps of lerps: blend neighbouring control points, blend those results, and keep going
until a single point remains — that point, swept over t in [0,1], IS the curve.
</Callout>

## Intuition: lerp, then lerp again

Linear interpolation between two points is the atom:

$$
\operatorname{lerp}(A, B, t) = (1-t)\,A + t\,B, \qquad t \in [0,1].
$$

At t=0 you get A, at t=1 you get B, and in between you glide straight across. Now stack it. Take three
points and lerp the first pair and the last pair, then lerp *those* two results — you have traced a
**quadratic Bézier**. Add a fourth point and recurse one level deeper and you have a **cubic Bézier**,
the workhorse of fonts, vector art and easing curves. This nested-lerp recipe is **de Casteljau's
algorithm**: geometric, numerically gentle (it only ever takes convex combinations, so it never
amplifies floating-point error), and the exact routine production engines like THREE.js run.

A Bézier curve is *pulled toward* its interior control points but passes only through the two endpoints —
the control points steer, they are not waypoints. When you genuinely need the curve to hit every point
(camera waypoints, animation keyframes), you reach for a **Catmull-Rom spline**, which threads all of its
points and stays smooth (C1 continuous) by choosing each point's tangent from its neighbours.

## How it works

Take a cubic Bézier with control points P0 = (0, 0), P1 = (0, 2), P2 = (2, 2), P3 = (2, 0) and evaluate at
t = 0.5. De Casteljau is nothing but repeated lerps, shrinking the array by one each round:

```
level 0 (control points):  P0(0,0)  P1(0,2)  P2(2,2)  P3(2,0)

level 1 — lerp each edge at t=0.5:
  Q0 = lerp(P0,P1,.5) = (0, 1)
  Q1 = lerp(P1,P2,.5) = (1, 2)
  Q2 = lerp(P2,P3,.5) = (2, 1)

level 2 — lerp the 2 remaining edges:
  R0 = lerp(Q0,Q1,.5) = (0.5, 1.5)
  R1 = lerp(Q1,Q2,.5) = (1.5, 1.5)

level 3 — the last lerp:
  B  = lerp(R0,R1,.5) = (1.0, 1.5)   ← the point on the curve
```

Three points became one. That final `(1.0, 1.5)` is B(0.5), and no polynomial coefficients were ever
computed — only convex combinations of points. Here is the same construction as runnable code:

      next.push(lerp(pts[i], pts[i + 1], t));

    pts = next;

## The real implementation

The course file evaluates a Bézier two ways — the numerically stable de Casteljau recursion and the
direct Bernstein sum — and adds Catmull-Rom for curves that must pass through their points:

```ts
// /dsa/33-3d-graphics/07-curves-animation/curves-splines.ts
export function bezierDeCasteljau(points: Vec3[], t: number): Vec3 {
  let pts = points.map((p) => p.clone());
  while (pts.length > 1) {
    const next: Vec3[] = [];
    for (let i = 0; i < pts.length - 1; i++) {
      next.push(pts[i].lerp(pts[i + 1], t)); // (1−t)Pᵢ + t Pᵢ₊₁
    }
    pts = next;
  }
  return pts[0];
}

// Catmull-Rom segment p1→p2, tangents from neighbours: m₁=(p2−p0)/2, m₂=(p3−p1)/2.
export function catmullRomSegment(p0: Vec3, p1: Vec3, p2: Vec3, p3: Vec3, t: number): Vec3 {
  const t2 = t * t, t3 = t2 * t;
  const calc = (a: number, b: number, c: number, d: number): number =>
    0.5 * (2 * b + (-a + c) * t + (2 * a - 5 * b + 4 * c - d) * t2 + (-a + 3 * b - 3 * c + d) * t3);
  return new Vec3(
    calc(p0.x, p1.x, p2.x, p3.x),
    calc(p0.y, p1.y, p2.y, p3.y),
    calc(p0.z, p1.z, p2.z, p3.z),
  );
}
```

The sibling file `skeletal-animation.ts` reuses the very same interpolation atoms over *time*: a bone's
position and scale are **lerp**-ed between keyframes, and its rotation is **slerp**-ed (spherical lerp of
quaternions, so the turn advances at constant angular speed along the shortest arc). Keyframes are just
control points on a timeline; easing curves that shape the blend are — you guessed it — cubic Béziers.

## Complexity & math

A degree-3 Bézier expands into the **Bernstein basis** — the binomial weights of Pascal's triangle:

$$
B(t) = \sum_{i=0}^{3} \binom{3}{i} (1-t)^{3-i}\, t^{i}\, P_i
     = (1-t)^3 P_0 + 3(1-t)^2 t\, P_1 + 3(1-t) t^2 P_2 + t^3 P_3.
$$

The weights are non-negative and sum to one, $\sum_i \binom{3}{i}(1-t)^{3-i}t^i = ((1-t)+t)^3 = 1$, which
is why the curve stays inside the convex hull of its control points and why $B(0)=P_0$, $B(1)=P_3$. The
tangent (velocity) is the derivative $B'(t) = 3(1-t)^2 (P_1 - P_0) + 6(1-t) t\, (P_2 - P_1) + 3 t^2 (P_3 - P_2)$ —
normalize it to orient a camera or a car along the path.

De Casteljau evaluates the same point with repeated lerps in $O(n^2)$ time for degree $n$ (tiny, usually
$n=3$) while staying numerically stable. Catmull-Rom evaluates one segment in $O(1)$ and, crucially,
*passes through* each control point with matching tangents on both sides, giving C1 continuity.

| Method | Passes through points? | Evaluate one point | Continuity |
|--------|------------------------|--------------------|------------|
| Bézier (de Casteljau) | Endpoints only | $O(n^2)$ | C-inf within a segment |
| Bézier (Bernstein sum) | Endpoints only | $O(n)$ | C-inf within a segment |
| Catmull-Rom | Every point | $O(1)$ per segment | C1 across joints |

## Common pitfalls & interview tips

- **Control points are not waypoints (for Bézier).** The curve touches only P0 and P3; it is pulled
  toward P1 and P2. If someone needs the curve to hit every point, switch to Catmull-Rom or a natural
  spline.
- **Equal steps in t are NOT equal steps in distance.** A curve moves slowly then quickly for uniform Δt.
  For constant-velocity motion, build an arc-length table and remap t — otherwise your camera lurches.
- **Chain cubics; don't raise the degree.** A single high-degree Bézier wiggles and loses local control.
  Real paths are piecewise cubics joined with matched tangents.
- **Rotations need slerp, not lerp.** Linearly interpolating quaternion components drifts off the unit
  sphere and changes speed; slerp keeps constant angular velocity along the short arc.
- **Interview cue:** "smooth path through N points" almost always means *spline* — say "piecewise cubic,
  Catmull-Rom if it must pass through the points, and watch the t-vs-arc-length distinction."

2. **Easy:** For a cubic Bézier, what are B(0) and B(1) in terms of the control points, and why?

4. **Medium:** Evaluate the Bernstein weights $\binom{3}{i}(1-t)^{3-i}t^i$ at t=0.25 and confirm they sum

5. **Hard:** Given four keyframe rotations, explain why you would slerp between them rather than lerp their
   raw quaternion components, and what "shortest arc" correction slerp applies.
