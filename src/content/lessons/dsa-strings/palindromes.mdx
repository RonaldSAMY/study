## The hook: a word that looks in a mirror

Stand `racecar` in front of a mirror and it looks identical to itself. So do `level`, `noon`, and the whole
phrase *"A man, a plan, a canal: Panama"*. A palindrome is a sequence with mirror symmetry, and the cleanest
way to test for it is to walk inward from both ends, checking that each pair of facing characters agrees.
The moment one pair disagrees, the mirror is broken and you can stop.

<Callout type="intuition" title="The one-sentence idea">
A string is a palindrome iff two pointers, starting at both ends and moving inward, see equal characters at
every step until they cross.
</Callout>

## Intuition: compare instead of swap

Last lesson the two pointers *swapped* to reverse. Here the very same converging pointers *compare*. Set
`left = 0`, `right = n - 1`; if `s[left] === s[right]`, step both inward; if they ever differ, return
`false` immediately. Cross the middle with no mismatch and the string is symmetric.

Why is this better than "reverse it and check equality"? Both are $O(n)$ time, but reversing allocates a
whole new string ($O(n)$ space), while the two-pointer scan keeps only two indices — $O(1)$ space — and it
**short-circuits**: a giant non-palindrome that fails on the first pair is detected instantly.

Two natural extensions build on the same symmetry:

- **Clean first, then compare.** Real palindrome puzzles ignore case, spaces, and punctuation, so normalize
  (lowercase, strip non-alphanumerics) before the scan.
- **Expand around center.** Every palindromic *substring* has a center. Stand at each of the $2n-1$
  possible centers (each character, and each gap between characters) and push two pointers *outward* while
  they match — that finds the longest palindromic substring in $O(n^2)$.

## How it works

Check `"racecar"` — the pointers meet in the middle with every pair equal:

```
r a c e c a r     L=0 R=6 : r == r ✓
r a c e c a r     L=1 R=5 : a == a ✓
r a c e c a r     L=2 R=4 : c == c ✓
r a c e c a r     L=3 R=3 : pointers meet → palindrome ✓
```

And the short-circuit on `"hello"`:

```
h e l l o         L=0 R=4 : h != o ✗  → return false (after ONE comparison)
```

  const s = clean ? str.toLowerCase().replace(/[^a-z0-9]/g, '') : str;
  let left = 0, right = s.length - 1;
  while (left < right) {

    left++; right--;
  }
  return true;

// Longest palindromic substring via expand-around-center
function longestPalindrome(s) {
  let best = '';
  const expand = (l, r) => {
    while (l >= 0 && r < s.length && s[l] === s[r]) { l--; r++; }
    return s.slice(l + 1, r);                  // widest match around this center
  };
  for (let i = 0; i < s.length; i++) {
    for (const cand of [expand(i, i), expand(i, i + 1)]) if (cand.length > best.length) best = cand;
  }
  return best;
}`}

## The real implementation

From the course file — the $O(1)$-space check and the expand-around-center search:

```ts
// /dsa/02-strings/palindrome.ts
function isPalindromeTwoPointer(str: string): boolean {
    let left = 0;
    let right = str.length - 1;
    while (left < right) {
        if (str[left] !== str[right]) return false;   // first mismatch ends it
        left++;
        right--;
    }
    return true;
}

function longestPalindromicSubstring(s: string) {
    let start = 0, maxLength = 1;
    const expandAroundCenter = (left: number, right: number): void => {
        while (left >= 0 && right < s.length && s[left] === s[right]) {
            if (right - left + 1 > maxLength) { start = left; maxLength = right - left + 1; }
            left--;
            right++;
        }
    };
    for (let i = 0; i < s.length; i++) {
        expandAroundCenter(i, i);      // odd-length center (a single char)
        expandAroundCenter(i, i + 1);  // even-length center (a gap)
    }
    return s.substring(start, start + maxLength);
}
```

## Complexity

The check is one converging scan; the longest-substring search tries every center:

$$
T_\text{check}(n) = O(n),\ S = O(1); \qquad T_\text{longest}(n) = O(n^2),\ S = O(1).
$$

| Task | Time | Space | Note |
|------|------|-------|------|
| Two-pointer check | $O(n)$ | $O(1)$ | short-circuits on first mismatch |
| Reverse-and-compare | $O(n)$ | $O(n)$ | allocates a copy, no early exit |
| Longest palindromic substring | $O(n^2)$ | $O(1)$ | $2n-1$ centers, expand each |
| Can-rearrange-to-palindrome | $O(n)$ | $O(k)$ | at most one odd character count |

## Common pitfalls & interview tips

- **Odd vs even centers.** A palindrome can have a single-character center (`aba`) or a gap center (`abba`).
  Expand-around-center must try **both** at every index or it misses all even-length palindromes.
- **Forgetting to normalize.** Phrase palindromes need case-folding and punctuation removal *first*; comparing
  raw characters will reject "A man a plan…".
- **Reverse-and-compare wastes space and never short-circuits.** Prefer the two-pointer scan unless you
  already need the reversed string for another reason.
- **Interview cue:** "valid palindrome", "longest palindromic substring", and "palindrome after removing one
  character" are all two-pointer problems — start by saying *"compare from both ends"*.
