## The hook: a wall you can read but never edit

A string is like a row of carved stone tiles. You can stand in front of any tile and read it instantly,
but you can't re-carve one in place — to "change" the wall you must build a brand-new wall. That single
fact, **immutability**, quietly decides why some string code is fast and some is accidentally quadratic.
Master the tile-wall mental model and most string algorithms become obvious.

<Callout type="intuition" title="The one-sentence idea">
A string is an immutable, zero-indexed sequence of character code units — random-access to read, but every
"edit" actually allocates a fresh string.
</Callout>

## Intuition: indexing, code units, and the cost of "+="

You already met arrays in the previous track: contiguous slots you can index in $O(1)$. A string is the
same shape — `s[i]` reads the character at position `i` in constant time — with one twist: you cannot
assign `s[i] = 'x'`. The characters are frozen.

Three consequences follow, and they explain almost every string pitfall:

1. **Reading is cheap, rewriting is not.** `s[i]` is $O(1)$, but producing a modified copy is $O(n)$.
2. **Building with `+=` in a loop is a trap.** Each `result += char` copies everything built so far, so a
   loop of length $n$ silently costs $O(n^2)$. Push into an array and `join('')` once instead.
3. **A "character" is a UTF-16 code unit.** For plain English that equals one letter, but emoji and many
   scripts span two code units (a *surrogate pair*), so `[...str]` is the Unicode-safe way to split.

The workhorse pattern for this lesson is the **frequency tally**: sweep the string once, counting how
often each character appears in a hash map. That single pass powers "first unique character", anagram
checks, and more — all in $O(n)$ time.

## How it works

Take `"letter"` and build the frequency map in one sweep, then scan once more for the first count of 1:

```
index:  0   1   2   3   4   5
char:   l   e   t   t   e   r
tally after the pass:  l:1  e:2  t:2  r:1
first char whose count is 1  →  'l' at index 0
```

Two linear passes, no nested loop. Here is that logic you can run:

## The real implementation

Straight from the course file — the frequency map and the first-unique scan, fully typed:

```ts
// /dsa/02-strings/string-basics.ts
function getCharFrequency(str: string): Map<string, number> {
    const freq = new Map<string, number>();
    for (const char of str) {
        freq.set(char, (freq.get(char) || 0) + 1);
    }
    return freq;
}

function firstNonRepeating(str: string): string | null {
    const count = new Map<string, number>();
    for (const char of str) {
        count.set(char, (count.get(char) || 0) + 1);   // pass 1: tally
    }
    for (const char of str) {
        if (count.get(char) === 1) return char;        // pass 2: first count of 1
    }
    return null;
}

// Unicode-safe split: spread handles surrogate pairs that str[i] would cut in half
function toCharArray(str: string): string[] {
    return [...str];
}
```

## Complexity

A frequency tally is one pass plus a hash map keyed by the alphabet:

$$
T(n) = O(n), \qquad S(n) = O(k),
$$

where $k$ is the number of *distinct* characters (at most the alphabet size, so $O(1)$ for ASCII).

| Operation | Time | Space | Note |
|-----------|------|-------|------|
| Index read `s[i]` | $O(1)$ | $O(1)$ | random access, like an array |
| Frequency map (one pass) | $O(n)$ | $O(k)$ | the core string trick |
| Build with `+=` in a loop | $O(n^2)$ | $O(n)$ | copies on every step — avoid |
| Build with array + `join` | $O(n)$ | $O(n)$ | the fix |

## Common pitfalls & interview tips

- **The `+=`-in-a-loop quadratic.** Because strings are immutable, each concatenation copies the prefix.
  For anything bigger than a few characters, collect parts in an array and `join('')` once.
- **`str.length` counts code units, not glyphs.** `"😀".length` is `2`. Use `[...str]` when you need true
  characters, and never split a string at an arbitrary `i` if emoji might be present.
- **`charCodeAt` vs `codePointAt`.** `charCodeAt` returns one UTF-16 unit; `codePointAt` returns the full
  Unicode code point. Reach for the latter when working beyond the Basic Multilingual Plane.
- **Interview cue:** the moment a question says "first unique", "all unique", or "counts of characters",
  say *"one frequency-map pass, O(n)"* — it's almost always the intended answer.

## Practice

1. **Easy:** By hand, build the frequency map of `"banana"`. Which character is the first non-repeating one?
2. **Easy:** Explain in one sentence why `let s=''; for (…) s += c;` over $n$ characters is $O(n^2)$.
3. **Medium:** Write `areAllUnique(str)` that returns `true` iff no character repeats, in one pass.
4. **Medium:** Given a string, return the character with the **highest** frequency (ties → the earliest).
5. **Hard:** Stream characters one at a time (you can't re-read the past) and after each one report the
   current first-unique character. Which data structures keep this $O(1)$ amortized per character?
