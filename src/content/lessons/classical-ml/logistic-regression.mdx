## The hook: should this email go to spam?

A filter can't answer *"spam or not"* with a shrug. But it also shouldn't pretend to be certain. What it
really wants to say is *"I'm 92% sure this is spam"* — a **probability** between 0 and 1. The same need
shows up everywhere a machine must make a yes/no call under uncertainty: a doctor weighing a diagnosis, a
bank sizing up a loan, an app deciding whether to show you an ad. Each one starts from a **score** — a
single number summarizing the evidence — and needs to turn that score into a calibrated chance. That
translation, from raw score to honest probability, is exactly what **logistic regression** does.

<Callout type="intuition" title="The one-sentence idea">
Logistic regression takes a **linear score** $w x + b$ and squashes it through the **sigmoid** into a
probability between 0 and 1, then learns $w$ and $b$ by making those probabilities match the labels.
</Callout>

## Intuition: from a straight line to a smooth switch

You already met **linear regression**: fit a line $w x + b$ to predict a number. The trouble is that a
line shoots off to $+\infty$ and $-\infty$, while a probability must stay politely between 0 and 1. So we
keep the familiar linear score — call it $z = w x + b$ — and then bend it.

The bending function is the **sigmoid**, an S-shaped curve. Feed it a big positive $z$ and it returns
something close to 1; feed it a big negative $z$ and you get close to 0; feed it exactly 0 and you land at
0.5 — the moment of maximum doubt. So logistic regression is really two familiar moves stacked: a linear
score (from regression), then a smooth squash into a probability.

Where the curve crosses $0.5$ is the **decision boundary** — the dividing point between predicting class 0
and class 1. Everything to one side leans one way; everything to the other side leans the other. Learning
means sliding and steepening that S so the boundary sits between your two groups of points.

## The formal definition

The **sigmoid** (logistic) function maps any real number into $(0, 1)$:

$$
\sigma(z) = \frac{1}{1 + e^{-z}}.
$$

We build the score linearly and pass it through the sigmoid to get the predicted probability of class 1:

$$
z = \mathbf{w} \cdot \mathbf{x} + b, \qquad p = \sigma(z) = \frac{1}{1 + e^{-(\mathbf{w}\cdot\mathbf{x} + b)}}.
$$

To train, we need a loss that punishes confident-but-wrong predictions. That loss is the **log-loss**, or
**binary cross-entropy**, averaged over $n$ examples with labels $y \in \{0, 1\}$:

$$
L = -\frac{1}{n} \sum_{i=1}^{n} \Big[\, y_i \log p_i + (1 - y_i)\log(1 - p_i) \,\Big].
$$

When the true label is $y = 1$ only the first term survives, so the loss is $-\log p$ — tiny when $p$ is
near 1, exploding when $p$ heads toward 0. The **decision boundary** is where the model is maximally
unsure, $p = 0.5$, which happens exactly when

$$
z = \mathbf{w}\cdot\mathbf{x} + b = 0.
$$

```python
# one forward pass + average log-loss
z = X @ w + b
p = 1 / (1 + np.exp(-z))
loss = -np.mean(y * np.log(p) + (1 - y) * np.log(1 - p))
```

<Callout type="why">
Why log-loss instead of the squared error from linear regression? Two reasons. First, it is
<strong>probabilistic</strong>: minimizing it is the same as maximizing the likelihood of the data under a
Bernoulli model, so the outputs behave like real probabilities. Second, paired with the sigmoid it is
<strong>convex</strong>, giving gradient descent a single bowl-shaped valley to roll into — squared error
on a sigmoid is bumpy and can trap the optimizer.
</Callout>

## The same idea, four different worlds

A linear score squashed into a probability shows up far beyond spam filters.

<ExampleCard domain="Healthcare" icon="🏥">
From a tumor's **size**, a model outputs the probability it is malignant. A small lump might score
$p = 0.08$ (likely benign), a large one $p = 0.94$ (likely malignant). The decision boundary is the size
at which the model tips from "watch" to "biopsy" — and a clinician can shift it to trade false alarms
against missed cases.
</ExampleCard>

<ExampleCard domain="Finance" icon="💳">
A lender turns an applicant's features into a score, then a **probability of default**. Output $p = 0.03$
and the loan is approved; output $p = 0.61$ and it is flagged. Because the answer is a calibrated chance,
the bank can price risk — higher predicted default, higher interest — instead of a blunt yes/no.
</ExampleCard>

<ExampleCard domain="Marketing" icon="📣">
Ad systems predict the **probability a user clicks**. Each ad-user pair gets a score; the sigmoid turns it
into a click-through probability like $p = 0.012$. Tiny numbers, but ranking by them decides which ad you
see — and multiplying by the bid is how the auction picks a winner.
</ExampleCard>

<ExampleCard domain="Sports" icon="🏀">
The **win probability** ticker on a live broadcast is logistic regression in action. From the score
margin and time remaining, the model squashes a linear score into a chance like $p = 0.78$ that the home
team wins — updating play by play as the evidence changes.
</ExampleCard>

<Callout type="note">
Notice the pattern: each domain feeds different features into the same machine. Build a score, squash it
into a probability, and pick a boundary. The features change; the math does not.
</Callout>
