## The hook: the block that ate deep learning

In 2017 one architecture quietly replaced almost everything — recurrent nets for language, and soon
convolutional nets for vision too. The transformer's secret isn't a single clever trick; it's a
**repeatable block** you can stack a hundred times without the training collapsing. Attention lets tokens
talk to each other; a feed-forward network lets each token think on its own; residual connections and
normalization keep the whole tower stable as it grows. Assemble those four pieces and you get GPT, BERT,
ViT, and the rest.

<Callout type="intuition" title="The one-sentence idea">
A transformer block is attention (tokens mix information) followed by a per-token MLP (each token
transforms itself), each wrapped in a residual connection and layer normalization so you can stack the
block deep.

</Callout>

## Intuition: mix, then think, on rails

You already know **attention** — the operation that lets every token pull in a weighted blend of the
others. A transformer wraps it in three supporting parts:

- **Positional encoding.** Attention alone is order-blind: shuffle the tokens and the outputs just
  shuffle too. Adding a position signal to each embedding tells the model *where* each token sits.
- **Feed-forward network (FFN).** After tokens exchange information, each one passes *independently*
  through a small two-layer MLP with a ReLU/GELU nonlinearity. This is where much of the model's raw
  capacity lives.
- **Residuals + normalization.** Each sub-layer computes $x + \text{Sublayer}(x)$ — a skip connection —
  then a LayerNorm rescales the result. The residual gives gradients a highway to flow down; the norm
  keeps activations from exploding or vanishing. Together they make 96-layer stacks trainable.

So one block is: **attention → add & norm → feed-forward → add & norm**. Stack $N$ of them, add an
embedding layer at the bottom and a prediction head at the top, and you have a transformer.

## How it works: the dry run

The FFN is the same little function applied to every position: $\text{FFN}(x) = \text{ReLU}(xW_1 + b_1)W_2 + b_2$,
usually with a hidden size $4\times$ the model dimension. The residual wrapper is what makes it safe:

```
sub-layer:  y = LayerNorm(x + Sublayer(x))
  x + Sublayer(x)   ← residual: keep the input, add the update
  LayerNorm(...)    ← rescale to mean 0, variance 1 (per token)

one block:  a = LayerNorm(x + Attention(x))     # tokens mix
            z = LayerNorm(a + FFN(a))            # tokens think
```

function layerNorm(v, eps = 1e-6) {
  const m = v.reduce((a, b) => a + b, 0) / v.length;
  const varr = v.reduce((a, b) => a + (b - m) ** 2, 0) / v.length;
  const sd = Math.sqrt(varr + eps);
  return v.map((x) => (x - m) / sd);
}
function relu(x) { return Math.max(0, x); }
// tiny FFN: hidden then back, plus residual + norm
function ffnBlock(x) {
  const hidden = x.map((v) => relu(v * 1.5 + 0.2));
  const out = hidden.map((h, i) => h * 0.8 - 0.1 + x[i]); // + residual
  return layerNorm(out);
}

## The real implementation

The course file assembles exactly this block — multi-head attention and FFN, each with a residual and a
LayerNorm:

```ts
// /dsa/27-ml-algorithms/02-transformers/transformer-basics.ts
class TransformerEncoderLayer {
  forward(X: Matrix, mask?: Matrix): Matrix {
    // 1. self-attention + residual + norm
    const attnOutput = this.multiHeadAttn.forward(X, X, X, mask);
    const norm1Output = this.norm1.forward(addMatrices(X, attnOutput));
    // 2. feed-forward + residual + norm
    const ffOutput = this.feedForward.forward(norm1Output);
    return this.norm2.forward(addMatrices(norm1Output, ffOutput));
  }
}
```

Run it with `npx tsx 27-ml-algorithms/02-transformers/transformer-basics.ts`. The sibling file
`positional-encoding.ts` shows the sinusoidal $PE(pos, 2i) = \sin(pos / 10000^{2i/d})$ that injects order.

## Complexity

For one block over a length-$n$ sequence with model dimension $d$:

$$
T(n) = O(\underbrace{n^2 d}_{\text{attention}} + \underbrace{n d^2}_{\text{FFN}}), \qquad S(n) = O(n^2 + nd).
$$

| Sub-layer | Time | Dominates when |
|-----------|------|----------------|
| Attention | $O(n^2 d)$ | long sequences ($n > d$) |
| Feed-forward | $O(n d^2)$ | short sequences ($n < d$) |

<Callout type="why">
Both terms matter in practice: for long documents the $n^2 d$ attention cost dominates, but for short
prompts the $n d^2$ feed-forward cost does — which is why FFN layers hold most of a model's parameters.
</Callout>

## Common pitfalls & interview tips

- **Dropping positional encoding.** Pure attention is permutation-equivariant, so "dog bites man" and
  "man bites dog" look identical without it.
- **Pre-norm vs post-norm.** Modern LLMs put LayerNorm *before* each sub-layer (pre-norm) for more stable
  deep training; the original paper used post-norm. Know the difference.
- **Residuals are not optional.** Remove the skip connections and gradients decay through the depth — the
  network becomes nearly untrainable past a handful of layers.
- **Interview cue:** "What are the four ingredients of a transformer block?" Attention, FFN, residual
  connections, and normalization — plus positional encoding at the input.
