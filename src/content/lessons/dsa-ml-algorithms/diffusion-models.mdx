## The hook: sculpting an image out of static

Stable Diffusion, DALL·E, and Midjourney all start from the same unpromising place: a screen of pure
random noise. Then, step by step, they *remove* noise until a coherent image appears — as if a sculptor
were chiselling a statue out of TV static. The astonishing part is how the model learned to do this: not
by studying how to draw, but by learning to **undo destruction**. Show it how images dissolve into noise,
and it learns to run the film backward.

<Callout type="intuition" title="The one-sentence idea">
Diffusion adds Gaussian noise to data in many small steps (fixed, easy), trains a network to predict and
remove that noise, then generates by starting from pure noise and denoising step by step.
</Callout>

## Intuition: a forward you can't learn, a reverse you can

There are two processes:

- **Forward (fixed, no learning).** Take a clean image $x_0$ and repeatedly sprinkle in a little Gaussian
  noise. After $T$ steps you have $x_T$ — indistinguishable from pure static. This direction needs no
  neural network; it's just a known noise schedule.
- **Reverse (learned).** Train a network to look at a noisy $x_t$ and predict the noise that was added.
  Subtract that prediction and you've taken one step back toward clean data. Chain $T$ of these and you
  walk from noise all the way to a fresh sample.

The clever shortcut: you never have to run the forward process one slow step at a time during training.
A closed form jumps straight to any step $t$:
$x_t = \sqrt{\bar{\alpha}_t}\,x_0 + \sqrt{1 - \bar{\alpha}_t}\,\varepsilon$, where $\bar{\alpha}_t$ is the
cumulative product of "how much signal survives." Sample a random $t$, noise the image, and ask the
network to recover $\varepsilon$. That's the entire training loop.

## How it works: the dry run

Training minimizes a shockingly simple objective — the mean-squared error between the true noise and the
network's prediction: $\mathcal{L} = \lVert \varepsilon - \varepsilon_\theta(x_t, t) \rVert^2$.

```
one training step:
  1. x0  ← a real example
  2. t   ← random step in 1..T
  3. ε   ← Gaussian noise, same shape as x0
  4. xt  = √ᾱ_t · x0 + √(1-ᾱ_t) · ε        # jump straight to step t
  5. predict ε̂ = model(xt, t)
  6. loss = mean( (ε - ε̂)² )               # just predict the noise!
```

## The real implementation

The course file precomputes the schedule so any step is one lookup, then noises with the closed form:

```ts
// /dsa/27-ml-algorithms/04-diffusion-models/diffusion-basics.ts
// forward: q(x_t | x_0) = √ᾱ_t · x_0 + √(1-ᾱ_t) · ε
qSample(x0: number[], t: number, noise?: number[]): number[] {
  noise = noise ?? GaussianUtils.randn([x0.length]);
  const a  = this.schedule.sqrtAlphasCumProd[t];         // √ᾱ_t
  const om = this.schedule.sqrtOneMinusAlphasCumProd[t]; // √(1-ᾱ_t)
  return x0.map((v, i) => a * v + om * noise![i]);
}
```

Run it with `npx tsx 27-ml-algorithms/04-diffusion-models/diffusion-basics.ts`. The sibling `ddim.ts`
shows how to *skip* steps at sampling time — turning 1000 denoising passes into ~50 without retraining.

## Complexity

Sampling is the bottleneck: each of $T$ steps is a full network pass.

$$
T_{\text{sample}} = O(T \cdot C), \qquad T_{\text{train per step}} = O(C),
$$

where $C$ is the cost of one model forward pass.

| Method | Steps to sample | Trade-off |
|--------|-----------------|-----------|
| DDPM | ~1000 | Highest quality, slowest |
| DDIM | ~20–50 | Nearly as good, ~20× faster |
| Distilled | 1–4 | Fastest, some quality loss |

<Callout type="why">
Unlike a GAN or a transformer that generates in one shot, diffusion trades compute for quality: more
denoising steps means sharper samples. Cutting that step count (DDIM, distillation, consistency models)
is the central research race in fast image generation.
</Callout>

## The same idea, different worlds

<ExampleCard domain="Photo restoration" icon="🖼️">
Denoising an old, grainy photograph is literally one reverse-diffusion step: estimate the noise
corrupting the image and subtract it. Diffusion generalizes this to removing *all* the noise, from pure
static.
</ExampleCard>

<ExampleCard domain="Audio & music" icon="🎵">
The same recipe generates speech and music: noise a waveform (or spectrogram), learn to denoise it.
Models like AudioLDM produce sound effects and songs by denoising exactly as image models denoise
pixels.
</ExampleCard>

<ExampleCard domain="Molecule design" icon="⚗️">
Diffusion runs on 3D atom coordinates: noise a molecule's structure, then denoise to sample new,
physically plausible shapes. Tools like RFdiffusion design novel proteins this way.
</ExampleCard>

## Common pitfalls & interview tips

- **Thinking the forward process is learned.** It isn't — noising is fixed math. Only the reverse
  denoiser has parameters.
- **Confusing what the network predicts.** Standard DDPM predicts the *noise* $\varepsilon$, not the clean
  image directly (though the two are equivalent via the closed form).
- **Too few sampling steps in plain DDPM.** Skipping steps naively degrades quality; use DDIM's
  deterministic sampler if you want speed.
- **Interview cue:** "Why do diffusion models train more stably than GANs?" No adversarial min-max game —
  it's a plain regression (predict the noise) with a well-behaved MSE loss.
