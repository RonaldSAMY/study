## The hook: you are the company you keep

To guess whether a social-media account is a bot, one strong clue is who it connects to: bots cluster
with bots. A molecule's properties depend on how its atoms are *bonded*, not just which atoms are
present. A fraudster hides in a web of transactions. All of these live on **graphs**, where the
structure — who is connected to whom — carries the signal. A Graph Neural Network learns from that
structure by having each node repeatedly **listen to its neighbors** and update itself.

<Callout type="intuition" title="The one-sentence idea">
A GNN updates every node by aggregating a normalized average of its neighbors' features, and repeating
that message-passing step lets information flow outward across the graph.
</Callout>

## Intuition: message passing

You already know graphs from adjacency lists and BFS. A GNN adds *features* to each node — a vector —
and one simple rule that fires in rounds:

1. **Message.** Every node looks at each neighbor's current feature vector.
2. **Aggregate.** It combines those messages (sum, mean, or max) with its own.
3. **Update.** It transforms the aggregate through a learned weight matrix and a nonlinearity to get its
   new feature.

After **one** round, each node knows about its immediate neighbors — like one hop of BFS. After **two**
rounds, it indirectly hears from neighbors-of-neighbors. After $k$ rounds, information has travelled $k$
hops. The famous Graph Convolutional Network (GCN) is just this with a specific *normalized* average that
stops high-degree hub nodes from dominating.

<Callout type="tip">
Run too many rounds and every node converges to the same value — <strong>over-smoothing</strong>. It's
why most GNNs are shallow (2–3 layers): unlike transformers, stacking many message-passing layers
*erases* the distinctions you're trying to learn.

## How it works: the dry run

A GCN layer is $H^{(l+1)} = \sigma\!\left(\tilde{D}^{-1/2}\tilde{A}\,\tilde{D}^{-1/2} H^{(l)} W^{(l)}\right)$,
where $\tilde{A} = A + I$ adds self-loops and $\tilde{D}$ is its degree matrix. Per node that's a
degree-weighted average of neighbors, then a linear map and ReLU:

```
node i update:
  h_i ← σ( Σ_{j ∈ N(i) ∪ {i}}  (1 / √(d_i · d_j)) · h_j · W )

  the 1/√(d_i·d_j) factor down-weights messages from
  high-degree neighbors so hubs don't drown everyone out.
```

const N = 4;

for (let i=0;i<N;i++) adj[i][i]=1;                 // self-loops
for (const [u,v] of edges){ adj[u][v]=1; adj[v][u]=1; }
const deg = adj.map((r)=>r.reduce((a,b)=>a+b,0));
function propagate(h) {
  return h.map((_, i) => {
    let acc = 0;
    for (let j=0;j<N;j++) if (adj[i][j]) acc += h[j] / Math.sqrt(deg[i]*deg[j]);
    return acc;

}
let h = [1, 0, 0, 0];         // a signal on node 0

## The real implementation

The course file builds the normalized adjacency once, then each layer is a matrix multiply through it:

```ts
// /dsa/27-ml-algorithms/03-graph-neural-networks/gcn.ts
// H^(l+1) = σ( D^(-1/2) A D^(-1/2) H^(l) W^(l) )
forward(features: Matrix, normalizedAdj: Matrix): Matrix {
  const hw  = features.multiply(this.weights);   // H · W   (transform)
  const ahw = normalizedAdj.multiply(hw);        // Â · (HW) (aggregate neighbors)
  // add bias, then apply the activation (ReLU)
  return ahw.addBias(this.bias).apply(this.activation);
}
```

Run it with `npx tsx 27-ml-algorithms/03-graph-neural-networks/gcn.ts`. The siblings `graphsage.ts`
(neighbor *sampling* for huge graphs) and `gat.ts` (attention *over* neighbors) are variations on the
same aggregate-and-update theme.

## Complexity

Work is proportional to the edges, since messages travel along them:

$$
T = O(L \cdot (E\,d + V\,d^2)), \qquad S = O(V d + E),
$$

for $L$ layers, $V$ nodes, $E$ edges, feature dimension $d$.

| Term | Cost | Meaning |
|------|------|---------|
| Aggregate ($\hat{A}H$) | $O(E\,d)$ per layer | one message per edge |
| Transform ($HW$) | $O(V\,d^2)$ per layer | learned linear map |

<Callout type="why">
Because cost scales with $E$, not $V^2$, GNNs handle sparse real-world graphs (social networks,
molecules) efficiently — you only ever touch edges that exist, never the full adjacency matrix.
</Callout>

## The same idea, different worlds

<ExampleCard domain="Chemistry" icon="🧪">
Represent a molecule as atoms (nodes) and bonds (edges). Message passing lets each atom sense its
chemical environment, predicting properties like toxicity or solubility — this is how modern
drug-discovery models screen candidates.
</ExampleCard>

<ExampleCard domain="Recommendations" icon="🛒">
Model users and products as a bipartite graph. A user's embedding aggregates the items they bought, and
each item aggregates its buyers — Pinterest and others use GraphSAGE this way to recommend at scale.
</ExampleCard>

<ExampleCard domain="Traffic & maps" icon="🚦">
Road intersections are nodes, roads are edges. A GNN propagates congestion from one junction to its
neighbors, which is how Google Maps improved arrival-time predictions across a city's road graph.
</ExampleCard>

## Common pitfalls & interview tips

- **Over-smoothing.** Too many layers and all node features converge — keep GNNs shallow (2–3 layers) or
  use residual/jumping-knowledge connections.
- **Forgetting self-loops.** Without $A + I$, a node's update ignores its *own* current feature and only
  sees neighbors. Add the identity.
- **Skipping normalization.** Plain neighbor sums let hub nodes explode in magnitude; the
  $\tilde{D}^{-1/2}\tilde{A}\tilde{D}^{-1/2}$ symmetric normalization fixes it.
- **Interview cue:** "How is a GNN related to a transformer?" Both aggregate information from other
  elements — a transformer is (loosely) attention over a *fully connected* graph; a GNN respects the
  actual edges.
