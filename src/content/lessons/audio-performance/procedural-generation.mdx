## The hook: a whole galaxy in a few kilobytes

A famous space game ships **18 quintillion** planets, yet the download is tiny. There is no giant file
of planets anywhere — each world is *computed on demand* from a number. Visit the same coordinates and
you get the same planet, atom for atom, every time. That magic is **procedural generation**: trading
storage for computation by turning a tiny **seed** into vast, reproducible content.

<Callout type="intuition" title="The one-sentence idea">
Feed a deterministic random-number generator one **seed** and apply rules to its output, and you get
an entire world that is both *varied* and *perfectly reproducible* from that single number.
</Callout>

## Intuition: controlled randomness

You already met **PRNGs** and **noise** in game math. Procedural generation is what you build on top of
them. The key word is **deterministic**: a pseudo-random generator is not really random — it is a fixed
recipe that, from the same seed, always produces the same stream of numbers. That is a feature, not a
bug:

- **Reproducibility** — the same seed rebuilds the same world, so you can store a sprawling level as
  one integer.
- **Variety** — change the seed and you get a completely different, still-coherent world.

There are three classic tools, in rising order of structure:

1. **Raw PRNG** — scatter things: loot, enemy placement, star positions.
2. **Noise** (value or Perlin) — smooth, natural gradients: terrain height, cave density, cloud cover.
3. **Grammars and rules** — constrained structure: rooms joined by corridors, buildings, road
   networks. Pure randomness gives chaos; rules give a place that makes sense.

## The formal definition

A PRNG is a recurrence that maps an internal state $s_n$ to the next state and an output:

$$
s_{n+1} = f(s_n), \qquad u_n = g(s_{n+1}) \in [0, 1).
$$

The starting state $s_0$ is the **seed**, so the whole stream $u_0, u_1, u_2, \dots$ is a deterministic
function of the seed. **Value noise** turns scattered random values on a lattice into a smooth field by
interpolating between them. For a point with fractional offset $t$ between two lattice values $a$ and
$b$, a smoothstep blend avoids hard edges:

$$
\operatorname{smooth}(t) = t^2 (3 - 2t), \qquad h = a + (b - a)\,\operatorname{smooth}(t).
$$

To place non-overlapping rooms, you sample candidate rectangles and keep one only if it stays clear of
every accepted room — rejection sampling under a constraint:

```python
rng = seeded_rng(seed)            # same seed -> same stream
rooms = []
for _ in range(attempts):
    r = random_rect(rng)
    if all(not overlaps(r, q) for q in rooms):
        rooms.append(r)           # accept; else reject and retry
connect_in_order(rooms)           # L-shaped corridors between centres
```

<Callout type="why">
Determinism is what makes seeds shareable and multiplayer-safe: if every client runs the same generator
on the same seed, every machine builds the identical world without sending the world over the network —
only the seed travels.
</Callout>

## The same idea, four different worlds

<ExampleCard domain="Roguelikes" icon="🗝️">
A roguelike rebuilds its dungeon from a fresh seed every run, so no two descents are alike — yet a
"daily challenge" hands every player the <strong>same</strong> seed so scores are comparable. One
integer is both infinite variety and a level playing field.
</ExampleCard>

<ExampleCard domain="Open-world survival" icon="🌲">
Survival-craft games generate terrain from layered noise: low-frequency noise carves continents,
higher-frequency noise adds hills and rivers. Players swap <strong>seed strings</strong> like recipes
because a single number reproduces a beloved spawn for everyone.
</ExampleCard>

<ExampleCard domain="Racing & mobile" icon="🏁">
An endless runner generates the track ahead in chunks as you go, seeded so difficulty ramps smoothly.
Nothing far behind is kept in memory — it can always be regenerated from its chunk seed, which is
ideal for a phone's tiny memory budget.
</ExampleCard>

<ExampleCard domain="City builders" icon="🏙️">
Procedural cities use a <strong>grammar</strong>: a road splits into smaller roads, blocks fill with
buildings sized by rules. The grammar guarantees streets actually connect, so the result looks
designed, not random — structure on top of the noise.
</ExampleCard>
