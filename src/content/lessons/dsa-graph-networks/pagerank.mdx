## The hook: how do you rank the whole web with no editors?

In 1998 the web had hundreds of millions of pages and no one to grade them. Counting inbound links was
the obvious idea — but spammers can mint a million junk pages that all link to theirs. The insight that
launched Google was to make a link's value depend on the *importance of the page casting it*. A vote from
a page that is itself heavily linked counts more than a vote from a nobody. That is circular — your rank
depends on your neighbours' ranks, which depend on yours — and resolving that circle is exactly PageRank.

<Callout type="intuition" title="The one-sentence idea">
Imagine a surfer who clicks random links forever; PageRank is the fraction of time they spend on each
page — high if many important pages point to it.
</Callout>

## Intuition: a random surfer who sometimes teleports

Picture someone browsing forever. From the current page they do one of two things:

- With probability $d$ (the **damping factor**, usually $0.85$) they follow a random out-link.
- With probability $1 - d$ they get bored and **teleport** to a page chosen uniformly at random.

Run this walk for a long time and the surfer settles into a steady state: each page is visited a fixed
fraction of the time. That fraction is its PageRank. Teleporting is not a detail — it guarantees the walk
can always escape dead ends and link farms, so a unique steady state always exists.

The circularity resolves by **iteration**. Start every page equal, then repeatedly let each page hand its
score out evenly along its links. Importance flows along edges, pools at well-connected pages, and after a
dozen or so rounds the numbers stop moving. That fixed point is the answer.

## How it works

Each iteration applies one rule to every page at once: a page's new score is a small teleport share plus
the damped sum of scores arriving along its in-links. For a page $A$ with the pages $i$ linking to it:

$$
PR(A) = \frac{1 - d}{N} + d \sum_{i \to A} \frac{PR(i)}{L(i)},
$$

where $N$ is the number of pages, $L(i)$ is the number of out-links from page $i$, and $d$ is the damping
factor. Every page divides its score equally among the pages it points to — a page with many out-links
gives each neighbour only a thin slice.

Watch three pages where $B \to A$, $C \to A$, and $A \to C$ converge from a flat start:

```
iter 0:  A=0.333  B=0.333  C=0.333
iter 1:  A=0.473  B=0.050  C=0.477
iter 2:  A=0.457  B=0.050  C=0.493
iter 3:  A=0.461  B=0.050  C=0.489   ← barely moving: converged
```

B linked to nothing important and was linked to by nothing, so it sinks to the teleport floor; A and C
trade rank back and forth and settle.

function pageRank(edges, nodes, d = 0.85, iters = 50) {
  const out = {}, inc = {};
  nodes.forEach(n => { out[n] = []; inc[n] = []; });
  for (const [u, v] of edges) { out[u].push(v); inc[v].push(u); }
  const N = nodes.length;
  let pr = {}; nodes.forEach(n => pr[n] = 1 / N);
  for (let it = 0; it < iters; it++) {
    const next = {};
    let dangling = 0;
    nodes.forEach(n => { if (out[n].length === 0) dangling += pr[n]; });
    for (const v of nodes) {
      let s = 0;
      for (const u of inc[v]) s += pr[u] / out[u].length;
      next[v] = (1 - d) / N + d * (s + dangling / N);
    }
    pr = next;
  }
  // round for readability
  const r = {}; nodes.forEach(n => r[n] = +pr[n].toFixed(3));
  return r;
}`}

## The real implementation

The course file wraps power iteration in a class and adds the personalized variant — instead of teleporting
uniformly, you teleport into a chosen set of pages, which biases the ranking toward one user's interests.

```ts
// /dsa/29-graph-advanced/pagerank-advanced.ts
computePageRank(iterations: number = 100): Map<number, number> {
  const pr = new Map<number, number>();
  const N = this.nodes.length;
  this.nodes.forEach(node => pr.set(node, 1 / N));      // start flat

  for (let iter = 0; iter < iterations; iter++) {
    const newPr = new Map<number, number>();
    for (const node of this.nodes) {
      let sum = 0;
      for (const [from, targets] of this.graph) {        // incoming links
        if (targets.includes(node)) {
          sum += (pr.get(from) || 0) / targets.length;   // share of source's rank
        }
      }
      newPr.set(node, (1 - this.dampingFactor) / N + this.dampingFactor * sum);
    }
    pr.clear();
    for (const [node, value] of newPr) pr.set(node, value);
  }
  return pr;
}
```

## Complexity

Each iteration touches every edge once to push rank, and the walk converges geometrically — the error
shrinks by a factor of $d$ each round, so a fixed tolerance needs only $O(\log(1/\varepsilon))$ iterations,
independent of graph size:

$$
T = O\!\left(k \,(N + M)\right), \qquad S = O(N + M),
$$

where $k$ is the iteration count, $N$ the nodes, and $M$ the edges.

| Quantity | Cost | Note |
|----------|------|------|
| One iteration | $O(N + M)$ | every node and edge once |
| Iterations to converge | $O(\log(1/\varepsilon))$ | $\approx 50$–$100$ in practice |
| Memory | $O(N + M)$ | the graph plus two rank vectors |

<Callout type="why">
PageRank is an <strong>eigenvector</strong> problem in disguise: the converged scores are the dominant
eigenvector of the (damped) link matrix, and power iteration is the standard way to find it. If you have
met the idea in the ML course's [eigenvalues &amp; eigenvectors lesson](/lessons/linear-algebra/eigenvectors),
PageRank is its most famous application — the web's link matrix, its principal eigenvector read off as
importance.
</Callout>

## Common pitfalls & interview tips

- **Dangling nodes leak rank.** A page with no out-links has nowhere to send its score. Redistribute it
  (uniformly, as the demo does) or the ranks stop summing to one.
- **Forgetting teleport breaks convergence.** Without the $1 - d$ term a rank sink (a cycle with no way out)
  swallows all the score. Damping guarantees a unique steady state.
- **It is global, not local.** PageRank cannot be read off one node's neighbourhood — it depends on the
  whole graph, which is why you iterate to a fixed point.
- **Interview cue:** "rank nodes by importance where importance is recursive" almost always means PageRank
  or power iteration. Say "dominant eigenvector of the transition matrix" and explain the random surfer.
