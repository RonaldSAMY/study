## The hook: find the bridges, then burn them

Louvain builds communities bottom-up by merging. Girvan-Newman does the opposite: it finds the few edges
that *bridge* communities and snips them, letting the network fall apart into its natural clusters. The
clever part is spotting the bridges. A road connecting two towns carries traffic between every pair of
people on opposite sides — so it is the busiest road per capita. The graph version of "busiest" is **edge
betweenness**: how many shortest paths run through an edge. Bridges between communities have sky-high
betweenness, so removing the highest-betweenness edge again and again peels communities apart.

<Callout type="intuition" title="The one-sentence idea">
The edge with the most shortest paths crossing it is the bridge between groups, so cutting the
highest-betweenness edge repeatedly splits the network into communities.
</Callout>

## Intuition: betweenness reveals the seams

Within a tight community there are many short ways to get from one node to another, so no single internal
edge is essential. But to cross from one community to another, almost every shortest path is funnelled
through the handful of edges that connect them. Those bottleneck edges therefore accumulate enormous
betweenness — they sit on the shortest path between most cross-community pairs.

Girvan-Newman exploits this directly:

1. Compute edge betweenness for every edge (run shortest paths between all node pairs, tally edge usage).
2. Remove the single edge with the highest betweenness.
3. Recompute betweenness — it changes once an edge is gone — and repeat.

Each removal eventually disconnects a piece, and the sequence of splits builds a hierarchy of communities
from coarse to fine. It is accurate and intuitive, but recomputing all-pairs shortest paths after every cut
makes it expensive — the price of its top-down clarity.

## How it works

The **betweenness** of edge $e$ sums, over all node pairs $(s,t)$, the fraction of shortest $s$–$t$ paths
that use $e$:

$$
C_B(e) = \sum_{s \neq t} \frac{\sigma_{st}(e)}{\sigma_{st}},
$$

where $\sigma_{st}$ is the number of shortest paths from $s$ to $t$ and $\sigma_{st}(e)$ is how many of them
pass through $e$. (When several shortest paths exist, each edge gets a fractional share, which is why the
formula divides by $\sigma_{st}$.) The outer loop is short:

```
repeat until enough communities:
  compute C_B(e) for every edge        ← BFS shortest paths from every node
  remove edge with the largest C_B
  recompute (the graph changed)
connected components = communities
```

Trace two triangles joined by a single bridge $C-D$. Every shortest path from the A-B-C triangle to the
D-E-F triangle must use $C-D$, so its betweenness dwarfs the triangle edges. Cut it, and the graph splits
cleanly into the two triangles.

function girvanNewmanStep(edges, nodes) {
  const adj = {}; nodes.forEach(n => adj[n] = new Set());
  for (const [a, b] of edges) { adj[a].add(b); adj[b].add(a); }
  const key = (u, v) => u < v ? u + '-' + v : v + '-' + u;
  // all shortest paths via BFS predecessors
  const shortest = (s, t) => {
    const dist = { [s]: 0 }, preds = { [s]: [] }, q = [s];
    while (q.length) { const u = q.shift();
      for (const v of adj[u]) {
        if (dist[v] === undefined) { dist[v] = dist[u] + 1; preds[v] = [u]; q.push(v); }
        else if (dist[v] === dist[u] + 1) preds[v].push(u);
      }
    }
    if (dist[t] === undefined) return [];
    const paths = [];
    const rec = (node, tail) => {
      if (node === s) { paths.push(tail); return; }
      for (const p of preds[node]) rec(p, [p, ...tail]);
    };
    rec(t, [t]); return paths;
  };
  const bw = {};
  for (let i = 0; i < nodes.length; i++) for (let j = i + 1; j < nodes.length; j++) {
    const paths = shortest(nodes[i], nodes[j]); if (!paths.length) continue;
    for (const p of paths) for (let k = 0; k < p.length - 1; k++) {
      const e = key(p[k], p[k + 1]); bw[e] = (bw[e] || 0) + 1 / paths.length;
    }
  }
  let cut = null, max = 0;
  for (const [e, v] of Object.entries(bw)) if (v > max) { max = v; cut = e; }
  return { cut, betweenness: +max.toFixed(2) };
}`}

## The real implementation

The course file calculates betweenness from all shortest paths, then removes the single strongest edge.

```ts
// /dsa/29-graph-advanced/girvan-newman.ts
removeHighestBetweennessEdge(): void {
  const betweenness = this.calculateEdgeBetweenness();
  let maxEdge = '';
  let maxScore = 0;
  for (const [edge, score] of betweenness) {
    if (score > maxScore) { maxScore = score; maxEdge = edge; }   // pick the busiest edge
  }
  if (maxEdge) {
    const [u, v] = maxEdge.split('-').map(Number);
    this.graph.get(u)?.delete(v);                                  // snip it both ways
    this.graph.get(v)?.delete(u);
  }
}
```

## Complexity

Computing betweenness from all-pairs shortest paths is the bottleneck, and it must be redone after every
one of the up-to-$m$ removals:

$$
T = O(m^{2} n), \qquad S = O(n + m),
$$

for $n$ nodes and $m$ edges (using Brandes' $O(mn)$ betweenness, repeated $O(m)$ times).

| Step | Cost |
|------|------|
| Edge betweenness (Brandes) | $O(mn)$ |
| Removals | up to $O(m)$ |
| Total | $O(m^{2} n)$ |

<Callout type="why">
Girvan-Newman is <strong>divisive</strong> (top-down splitting) where Louvain is <strong>agglomerative</strong>
(bottom-up merging). Seeing the same problem solved from opposite directions is a recurring ML theme —
hierarchical clustering does exactly this pair, and which direction wins depends on whether bridges or cores
are easier to spot in your data.
</Callout>

## Common pitfalls & interview tips

- **It is slow.** Recomputing all-pairs shortest paths after each cut makes Girvan-Newman impractical past a
  few thousand nodes — use it for clarity on small graphs, Louvain for scale.
- **Recompute, do not reuse.** Betweenness must be recalculated after every removal; the cut changes which
  paths are shortest, so cached scores go stale immediately.
- **When to stop is a choice.** The algorithm produces a whole hierarchy of splits; you pick the level
  (often by the partition with maximum modularity) — there is no built-in stopping rule.
- **Interview cue:** "split a network by cutting bridges" or "edge betweenness" is Girvan-Newman. Mention
  the divisive top-down nature and the $O(m^2 n)$ cost from repeated betweenness.

4. **Medium:** Sketch how you would stop Girvan-Newman at the partition with the highest modularity instead of cutting all edges.
5. **Hard:** Argue why, after removing one edge, only betweenness values for pairs whose shortest paths used that edge can change — and how that could speed up recomputation.
