## The hook: the skill tree

Open the character screen in almost any RPG and you'll see it: a **skill tree**. *Combat* branches into
*Melee* and *Magic*; *Melee* unlocks *Sword* and *Shield*. You can't grab *Sword* without first taking
*Melee*, which needs *Combat*. The data isn't a flat list or a lookup table — it's a **hierarchy**, where
each thing has a parent and may have children. That branching shape is a **tree**, and once you see it you
will find it everywhere: the folders holding your game's assets, the menu you're navigating, even the scene
of objects the engine draws each frame.

<Callout type="intuition" title="The one-sentence idea">
A **tree** is a set of nodes where each node has exactly one parent (except the top <strong>root</strong>)
and any number of children — a branching hierarchy with no loops.

</Callout>

## Intuition: family trees for data

Unlike an array (a flat row) or a hash map (an unordered pile), a tree captures *containment* and
*ancestry*. The vocabulary borrows straight from family trees:

- The **root** is the single node at the top, with no parent.
- A node's **children** are the nodes directly below it; their shared node above is the **parent**.
- A **leaf** is a node with no children — the end of a branch.
- The **depth** of a node is how many steps it is from the root.

A very common special case is the **binary tree**, where each node has at most two children (a "left" and a
"right"). It's the backbone of search trees, decision trees, and the expression trees a compiler builds.

Crucially, trees have **no cycles**: follow parents upward and you always reach the root, never a loop.
That guarantee is what makes them safe to walk recursively — and walking them is what we do next.

## The formal definition

A tree with $n$ nodes has exactly $n - 1$ edges (every node but the root is reached by one edge from its
parent). To process every node we **traverse** it. The four classic orders are:

- **Pre-order:** visit the node, then its children. *(root → left → right)*
- **In-order:** left child, node, right child — only meaningful for binary trees. *(left → root → right)*
- **Post-order:** children first, then the node. *(left → right → root)*
- **Level-order:** all depth-0 nodes, then depth-1, and so on — a breadth-first sweep.

Each visits all $n$ nodes exactly once, so every traversal is $O(n)$. The depth-first orders (pre, in,
post) are naturally **recursive** — they lean on the call **stack** from the stacks lesson — while
level-order uses an explicit **queue**.

<Callout type="why">
Trees are load-bearing in machine learning. A <strong>decision tree</strong> classifies data by walking
root-to-leaf, asking one question per node; random forests and gradient boosting stack hundreds of them.
And a <strong>scene graph</strong> — the tree of objects in a 3D world — is traversed every frame to figure
out where everything is drawn.
</Callout>

## The same idea, four different worlds

<ExampleCard domain="Biology" icon="🌳">
The tree of life is literally a tree: a common ancestor at the root branches into kingdoms, then species.
Tracing any two animals back to their shared ancestor is climbing to their lowest common parent.
</ExampleCard>

<ExampleCard domain="Company org chart" icon="🏢">
A CEO sits at the root; managers branch into teams branch into employees. Everyone has exactly one direct
boss (one parent), and the chain of command upward never loops — a textbook hierarchy.
</ExampleCard>

<ExampleCard domain="Cooking" icon="🍰">
A recipe breaks into sub-recipes: a cake needs a batter and an icing; the batter needs dry and wet mixes.
Making it bottom-up — finish each sub-mix before combining — is a <em>post-order</em> traversal in the
kitchen.
</ExampleCard>

<ExampleCard domain="Web pages" icon="🧩">
A web page is a tree of nested elements (the DOM): a page contains sections, which contain paragraphs,
which contain words. The browser walks this tree to lay everything out and draw it.
</ExampleCard>

<Callout type="note">
A tree is the strict, well-behaved cousin of the structure ahead. Loosen the rules — let a node have
several parents, allow cycles — and a tree becomes a <strong>graph</strong>, the most general structure of
all.
</Callout>
