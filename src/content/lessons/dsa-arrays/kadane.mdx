## The hook: the best stretch of a roller-coaster month

A trader's daily profit-and-loss swings between gains and losses, and you want the single best *consecutive*
streak — the run of days with the largest total. The sliding window from the previous lesson stalls here:
once negatives are in play, adding a day can *lower* the total, so "grow then shrink" loses its footing.
Kadane's algorithm replaces the window with one crisp question asked at every day: *is the running streak
still helping me, or should I cut my losses and start a new streak right here?*

<Callout type="intuition" title="The one-sentence idea">
Carry a running sum of the current streak; at each element, if the streak has gone negative it can only drag
you down, so throw it away and start fresh — the best running sum ever seen is the maximum subarray.
</Callout>

## Intuition: never carry a losing streak forward

The key realisation is local and greedy. Suppose you know the best subarray *ending exactly at* the previous
index has sum `current`. What is the best subarray ending at the next element `x`? You have only two
choices:

1. **Extend** the previous streak: `current + x`.
2. **Start fresh** at `x` alone: `x`.

You pick whichever is larger. And notice *why* you would ever start fresh: only when `current` is negative,
because a negative prefix can only shrink any sum it is glued to. So the rule is

$$
\text{current} = \max\bigl(x,\ \text{current} + x\bigr),
$$

and you keep a separate `best` that records the largest `current` you have ever seen. That `best` is the
answer. Unlike the sliding window, there is no left pointer to drag back — a losing streak is simply
abandoned the moment it turns against you, which is exactly what lets negatives be handled in one pass.

## How it works

Run Kadane on `[-2, 1, -3, 4, -1, 2, 1, -5, 4]`:

```
i  x    current = max(x, current+x)     best
0  -2   -2                              -2
1   1   max(1, -2+1=-1)   = 1   ↺reset   1
2  -3   max(-3, 1-3=-2)   = -2  extend   1
3   4   max(4, -2+4=2)    = 4   ↺reset   4
4  -1   max(-1, 4-1=3)    = 3   extend   4
5   2   max(2, 3+2=5)     = 5   extend   5
6   1   max(1, 5+1=6)     = 6   extend   6   ← best
7  -5   max(-5, 6-5=1)    = 1   extend   6
8   4   max(4, 1+4=5)     = 5   extend   6
```

The winner is `6`, the subarray `[4, -1, 2, 1]`. Here is the one-liner, plus the variant that also reports
*where* the best subarray sits:

  code={`// Kadane — maximum subarray SUM, O(n) time, O(1) space
function maxSubarray(nums) {
  let current = nums[0], best = nums[0];
  for (let i = 1; i < nums.length; i++) {
    current = Math.max(nums[i], current + nums[i]); // extend or restart
    best = Math.max(best, current);
  }
  return best;
}

// Same idea, also tracking the start/end indices
function maxSubarrayRange(nums) {
  let current = nums[0], best = nums[0];
  let start = 0, tempStart = 0, end = 0;
  for (let i = 1; i < nums.length; i++) {
    if (nums[i] > current + nums[i]) { current = nums[i]; tempStart = i; }
    else current = current + nums[i];
    if (current > best) { best = current; start = tempStart; end = i; }
  }
  return { best, start, end, sub: nums.slice(start, end + 1) };
}`}

## The real implementation

From the course file — the canonical Kadane, and `maxProductSubarray`, which tracks both a max *and* a min
because multiplying by a negative flips them:

```ts
// /dsa/01-arrays/kadane-algorithm.ts
function maxSubarrayKadane(nums: number[]): number {
    if (nums.length === 0) return 0;
    let currentSum = nums[0];
    let maxSum = nums[0];
    for (let i = 1; i < nums.length; i++) {
        currentSum = Math.max(nums[i], currentSum + nums[i]); // extend or start fresh
        maxSum = Math.max(maxSum, currentSum);
    }
    return maxSum;
}

function maxProductSubarray(nums: number[]): number {
    if (nums.length === 0) return 0;
    let maxProduct = nums[0], currentMax = nums[0], currentMin = nums[0];
    for (let i = 1; i < nums.length; i++) {
        const num = nums[i];
        if (num < 0) [currentMax, currentMin] = [currentMin, currentMax]; // negative flips roles
        currentMax = Math.max(num, currentMax * num);
        currentMin = Math.min(num, currentMin * num);
        maxProduct = Math.max(maxProduct, currentMax);
    }
    return maxProduct;
}
```

## Complexity

One pass, two scalars — you cannot do better than touching each element once:

$$
T(n) = O(n), \qquad S(n) = O(1).
$$

| Approach | Time | Space |
|----------|------|-------|
| Brute force (all subarrays) | $O(n^3)$ | $O(1)$ |
| Better brute (running inner sum) | $O(n^2)$ | $O(1)$ |
| **Kadane** | $O(n)$ | $O(1)$ |

Kadane is also a tiny dynamic program: `dp[i]` = best subarray sum *ending at* `i` equals
$\max(nums[i],\ dp[i-1] + nums[i])$, and the answer is the maximum over all `dp[i]`. The O(1) version just
keeps the one `dp` value it still needs.

## Common pitfalls & interview tips

- **Initialise to `nums[0]`, not `0`.** Seeding `best = 0` returns `0` for an all-negative array, but the
  correct answer is the largest (least negative) element. Start both `current` and `best` at `nums[0]`.
- **"At least one element."** Maximum subarray means non-empty; do not let an empty window of sum `0` sneak
  in unless the problem explicitly allows it.
- **Product is not sum.** For maximum *product*, track both the running max and min, because a negative
  number swaps which is which.
- **Circular arrays.** The best wrap-around subarray is `total − (minimum subarray)`; take the max of that
  and plain Kadane, with a guard for the all-negative case.
- **Interview cue:** "maximum/largest sum of a contiguous subarray" is the canonical Kadane prompt — state
  the extend-or-restart decision and that you seed from the first element.
