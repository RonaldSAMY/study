## The hook: the shortest burst that crosses the line

A smart meter logs your electricity use minute by minute, and you want the **shortest** stretch of consecutive
minutes whose total crosses some threshold — the briefest spike. Checking every possible stretch is O(n²).
Instead, imagine a stretchy window over the timeline: push its right edge out to add minutes until the total
is big enough, then pull its left edge in to throw away minutes you no longer need. The window breathes
across the data exactly once.

<Callout type="intuition" title="The one-sentence idea">
A sliding window is two same-direction pointers bounding a contiguous run; you expand the right edge to
include more and contract the left edge to drop what you do not need, answering subarray questions in one
O(n) pass.

</Callout>

## Intuition: two pointers, but chasing instead of converging

In the last lesson the pointers moved *toward* each other. A sliding window keeps them moving the *same*
direction, and the gap between them — the elements from `left` to `right` — is the window. Two flavours show
up constantly:

- **Fixed size:** the window is always `k` wide. Slide it one step by adding the new right element and
  subtracting the one that fell off the left. That is how a moving average is computed without re-summing.
- **Variable size:** grow `right` until the window satisfies a condition (sum reaches a target, or it still
  holds only distinct items), then shrink `left` while it stays satisfied, recording the best window as you
  go.

The magic is that each pointer only ever moves forward. Across the whole run `right` advances $n$ times and
`left` advances at most $n$ times, so even though the window resizes constantly, the total work is $O(n)$ —
not the $O(n \times k)$ you would pay by re-scanning each window from scratch.

## How it works

Find the shortest subarray of `[2, 3, 1, 2, 4, 3]` with sum at least `7`:

```
window [2]              sum 2  < 7   grow →
window [2,3]            sum 5  < 7   grow →
window [2,3,1]          sum 6  < 7   grow →
window [2,3,1,2]        sum 8 ≥ 7    length 4, shrink ← (drop 2)
window [3,1,2]          sum 6  < 7   grow →
window [3,1,2,4]        sum 10 ≥ 7   length 4, shrink ← (drop 3)
window [1,2,4]          sum 7 ≥ 7    length 3, shrink ← (drop 1)
window [2,4]            sum 6  < 7   grow →
window [2,4,3]          sum 9 ≥ 7    length 3, shrink ← (drop 2)
window [4,3]            sum 7 ≥ 7    length 2  ← best!
```

The answer is `2` (the window `[4, 3]`). Here is that variable window, plus the fixed-size moving maximum sum:

<CodeRunner
  client:visible
  code={`// Shortest subarray with sum ≥ target — variable window, O(n)
function minSubArrayLen(target, nums) {
  let left = 0, sum = 0, best = Infinity;
  for (let right = 0; right < nums.length; right++) {
    sum += nums[right];               // expand
    while (sum >= target) {           // contract while still valid
      best = Math.min(best, right - left + 1);
      sum -= nums[left];
      left++;
    }
  }
  return best === Infinity ? 0 : best;
}

// Max sum of any fixed window of size k — slide, don't re-sum, O(n)
function maxSumSubarray(arr, k) {
  let windowSum = 0;
  for (let i = 0; i < k; i++) windowSum += arr[i];
  let maxSum = windowSum;
  for (let i = k; i < arr.length; i++) {
    windowSum += arr[i] - arr[i - k]; // add new, drop old
    maxSum = Math.max(maxSum, windowSum);
  }
  return maxSum;
}`}

## The real implementation

From the course file — the variable window above, and the classic "longest substring without repeating
characters", which slides `left` past the previous occurrence of a repeat:

```ts
// /dsa/01-arrays/sliding-window.ts
function minSubArrayLen(target: number, nums: number[]): number {
    let left = 0, sum = 0, minLength = Infinity;
    for (let right = 0; right < nums.length; right++) {
        sum += nums[right];                                  // expand right
        while (sum >= target) {                              // shrink left while valid
            minLength = Math.min(minLength, right - left + 1);
            sum -= nums[left];
            left++;
        }
    }
    return minLength === Infinity ? 0 : minLength;
}

function lengthOfLongestSubstring(s: string): number {
    const charIndex = new Map<string, number>();
    let left = 0, maxLength = 0;
    for (let right = 0; right < s.length; right++) {
        const char = s[right];
        if (charIndex.has(char) && charIndex.get(char)! >= left) {
            left = charIndex.get(char)! + 1;                 // jump past the repeat
        }
        charIndex.set(char, right);
        maxLength = Math.max(maxLength, right - left + 1);
    }
    return maxLength;
}
```

## Complexity

`right` makes $n$ moves; `left` makes at most $n$ moves; nothing is revisited:

$$
T(n) = O(n), \qquad S(n) = O(1)\ \text{(fixed window)} \ \text{or}\ O(k)\ \text{(distinct-element window)}.
$$

| Approach | Time | Space |
|----------|------|-------|
| Recompute every subarray | $O(n^2)$ | $O(1)$ |
| Re-sum each fixed window | $O(n \cdot k)$ | $O(1)$ |
| **Sliding window** | $O(n)$ | $O(1)$–$O(k)$ |

The space is $O(1)$ when you track a single running sum, and $O(k)$ when the window must remember which
elements (or characters) it currently contains.

## Common pitfalls & interview tips

- **Variable windows need positive contributions.** The "grow until valid, then shrink" pattern relies on
  adding an element never *hurting* the condition. With negative numbers the window can shrink the sum, so
  `minSubArrayLen` assumes positives — use Kadane (next lesson) when negatives appear.
- **Off-by-one in the width.** The window length is `right - left + 1`, not `right - left`.
- **Shrink with `while`, not `if`.** After expanding once you may need to contract several times before the
  window is valid (or minimal) again.
- **Fixed vs variable.** "Subarray/substring of size k" → fixed window; "longest/shortest satisfying a
  condition" → variable window. Spotting which is half the battle.
- **Interview cue:** the words "contiguous", "subarray/substring", and "at most / at least K" almost always
  mean sliding window.
