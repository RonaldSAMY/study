## The hook: the moves every competitive coder has memorised

Once you know the six operators, a handful of *combinations* show up again and again — so often that
experienced programmers type them without thinking. "Is this a power of two?" is `n & (n - 1)`.
"Isolate the lowest set bit" is `n & -n`. These one-liners look like magic the first time you see
them, but each is just a mask meeting an operator at the right column.

<Callout type="intuition" title="The one-sentence idea">
Almost every bit trick is the same recipe: **build a mask** (usually `1 << i` or a clever neighbour
like `n - 1`) and **combine it** with the number using AND, OR, or XOR.
</Callout>

## Intuition: build a mask, pick an operator

A **mask** is a number whose bits mark the positions you care about. The simplest mask is `1 << i`: a
single 1 sitting at position $i$. Pair it with an operator and you control exactly one bit:

- **Get** bit $i$: `(n >> i) & 1` — slide it down and keep just the ones place.
- **Set** bit $i$: `n | (1 << i)` — OR forces it to 1.
- **Clear** bit $i$: `n & ~(1 << i)` — AND with everything-except-$i$ forces it to 0.
- **Toggle** bit $i$: `n ^ (1 << i)` — XOR flips it.

The two "wow" tricks use a *non-obvious* mask. Subtracting 1 from $n$ flips its lowest 1 and every
zero below it, which makes both of these fall out:

- **Isolate the lowest set bit:** `n & -n` (since $-n = \sim n + 1$).
- **Test power of two:** `n > 0 && (n & (n - 1)) === 0` — a power of two has exactly one 1-bit.

## How it works

Why does `n & (n - 1)` clear the lowest set bit? Take `n = 12 = 1100`. Then `n - 1 = 1011`: subtraction
borrows through the trailing zeros, flipping the lowest 1 to 0 and the zeros below it to 1.

```
  1100   (12)
& 1011   (11)
  ----
  1000   (8)   ← the lowest 1-bit is gone
```

If `n` had only one 1-bit to begin with (a power of two), there is nothing left, so the result is `0`.
That is the entire power-of-two test. And `n & -n` does the opposite — it *keeps* only that lowest bit,
because in two's complement `-n` is `~n + 1`, which agrees with `n` in exactly one position.

## The real implementation

Straight from the course file — the power-of-two test and the two rightmost-bit isolators that power
later lessons:

```ts
// /dsa/16-bit-manipulation/bit-basics.ts
function isPowerOfTwo(n: number): boolean {
    return n > 0 && (n & (n - 1)) === 0;   // exactly one 1-bit
}

function getRightmostSetBit(n: number): number {
    return n & (-n);                       // isolate the lowest 1-bit
}

function removeRightmostSetBit(n: number): number {
    return n & (n - 1);                    // clear the lowest 1-bit
}
```

Run the full file: `npx tsx 16-bit-manipulation/bit-basics.ts`.

## Complexity

Each trick is a fixed handful of operators on a machine word — no loops, no allocation:

$$
T(n) = O(1), \qquad S(n) = O(1).
$$

| Trick | Expression | Time |
|-------|------------|------|
| Get / Set / Clear / Toggle bit $i$ | `(n>>i)&1`, `n\|(1<<i)`, `n&~(1<<i)`, `n^(1<<i)` | $O(1)$ |
| Lowest set bit | `n & -n` | $O(1)$ |
| Power of two? | `n>0 && (n&(n-1))===0` | $O(1)$ |

## Common pitfalls & interview tips

- **Power-of-two needs the `n > 0` guard.** For `n = 0`, `n & (n - 1)` is also `0`, but zero is not a
  power of two. Forgetting the guard is the #1 review comment on this trick.
- **`1 << i` overflows past bit 31 in JS.** For positions ≥ 31, the result goes negative or wraps. Use
  `2 ** i` or `BigInt` when you need higher bits.
- **`-n` only behaves for signed integers.** It relies on two's complement; it does not mean "subtract"
  here, it means "isolate the lowest bit."
- **Interview cue:** "lowest set bit" and "power of two in $O(1)$" are warm-up questions that gate the
  harder ones — answer them instantly to buy credibility.

## Practice

1. **Easy:** Use `setBit` and `clearBit` to turn `0b1010` into `0b1110`, then back.
2. **Easy:** What is `lowestSetBit(40)`? Verify in the demo (`40 = 101000`).
3. **Medium:** Write `clearLowestSetBit` and use it to count set bits by looping until `n` is 0.
4. **Medium:** Given a power of two, find its exponent (which bit is set) using `Math.log2`.
5. **Hard:** Implement `isPowerOfFour(n)` — a power of two whose single bit sits at an *even* position
   (hint: AND with the mask `0x55555555`).
