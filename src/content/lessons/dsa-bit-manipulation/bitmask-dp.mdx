## The hook: an integer that *is* a set

Some problems force a brutal question: of $n$ items, which *subset* have I used so far? With $n = 15$
there are $2^{15}$ possible answers — far too many to track with named variables. But here is the
leap: a subset of up to ~20 items **is just an integer**. Bit $i$ on means "item $i$ is in." Suddenly
a DP table indexed by an integer remembers an answer for *every* subset, and famously hard problems
like the traveling salesman become tractable.

<Callout type="intuition" title="The one-sentence idea">
A bitmask turns a subset into a single integer, so `dp[mask]` can store the best answer for "having
already used exactly the items whose bits are set in `mask`."
</Callout>

## Intuition: counting from 0 to 2ⁿ − 1 visits every subset

Build on the [flags lesson](/lessons/dsa-bit-manipulation/flags-masks): each bit names one element. Now
let the integer *range* from `0` to `2^n - 1`. Every value in that range is a different combination of
on/off bits — so counting through all integers enumerates **all $2^n$ subsets, each exactly once**. The
empty set is `0`; the full set is `(1 << n) - 1`.

That gives DP a clean state space. The recipe is always:

- **State:** a `mask` of which items are done (often plus a "current position").
- **Transition:** pick an unused item `i` (`!(mask & (1 << i))`), move to `mask | (1 << i)`.
- **Answer:** read `dp[(1 << n) - 1]`, the cell for "everything used."

You'll also iterate **submasks** of a mask with the idiom `for (sub = mask; sub > 0; sub = (sub-1) & mask)`,
which walks every subset of the already-chosen items.

## Play with it

Define a small set and count the mask from 0 upward. Each integer lights up the elements whose bits are
set — that mask *is* the subset. The grid fills in as you go; by `2^n - 1` you've seen every combination
exactly once. That sweep is precisely the outer loop of a bitmask DP.

## How it works

The traveling salesman problem (TSP) is the canonical example. Let `dp[mask][i]` be the cheapest route
that visits exactly the cities in `mask` and ends at city `i`. To extend it, go to an unvisited city
`next` and turn its bit on:

```
dp[mask | (1<<next)][next] = min(
    dp[mask | (1<<next)][next],
    dp[mask][last] + dist[last][next]
)
```

You start at `dp[1][0] = 0` (only city 0 visited, sitting at city 0) and finish by closing the loop
back to the start from the full mask `(1 << n) - 1`. The whole tour search becomes a sweep over
$2^n \cdot n$ table cells.

function subsets(items) {
  const n = items.length, out = [];

    for (let i = 0; i < n; i++) if (mask & (1 << i)) subset.push(items[i]);
    out.push(subset);
  }
  return out;
}

// Walk every submask of a mask (used inside DP transitions)
function submasks(mask) {
  const out = [];
  for (let sub = mask; sub > 0; sub = (sub - 1) & mask) out.push(sub);
  out.push(0);
  return out;
}`}

## The real implementation

The course file's TSP solver — the textbook $O(2^n \cdot n^2)$ bitmask DP:

```ts
// /dsa/16-bit-manipulation/bitmask-dp.ts
function tsp(dist: number[][]): number {
    const n = dist.length;
    const ALL_VISITED = (1 << n) - 1;
    const dp: number[][] = Array(1 << n).fill(null).map(() => Array(n).fill(Infinity));
    dp[1][0] = 0;                              // start: only city 0 visited

    for (let mask = 1; mask < (1 << n); mask++) {
        for (let last = 0; last < n; last++) {
            if (!(mask & (1 << last)) || dp[mask][last] === Infinity) continue;
            for (let next = 0; next < n; next++) {
                if (mask & (1 << next)) continue;          // already visited
                const newMask = mask | (1 << next);
                dp[newMask][next] = Math.min(
                    dp[newMask][next],
                    dp[mask][last] + dist[last][next]
                );
            }
        }
    }
    let best = Infinity;                        // close the loop back to city 0
    for (let last = 1; last < n; last++) best = Math.min(best, dp[ALL_VISITED][last] + dist[last][0]);
    return best;
}
```

## Complexity

There are $2^n$ masks; TSP pairs each with a "last city" and tries every "next," giving:

$$
T_{\text{TSP}}(n) = O(2^n \cdot n^2), \qquad S(n) = O(2^n \cdot n).
$$

| Problem | Time | Space |
|---------|------|-------|
| Subset enumeration | $O(2^n \cdot n)$ | $O(1)$ extra |
| Submask enumeration (all masks) | $O(3^n)$ | $O(1)$ extra |
| TSP / Hamiltonian path | $O(2^n \cdot n^2)$ | $O(2^n \cdot n)$ |

Exponential — but for $n \le 20$, $2^{20} \approx 10^6$ states is perfectly fast. Beyond ~25, the table
explodes and you need a different technique.

## Common pitfalls & interview tips

- **Bitmask DP is only for small $n$.** The hard cap is roughly $n \le 20$–$22$. If $n$ can be large,
  it's the wrong tool — say so before coding.
- **`1 << n` overflows in JS for $n \ge 31$.** Stay well under that, or switch to `BigInt` for the
  table index arithmetic.
- **Initialise unreachable states to $\infty$ (or −1).** Skipping that lets garbage propagate into your
  minimum; guard every transition with "is this state reachable?"
- **The submask idiom is exact:** `sub = (sub - 1) & mask` visits each subset of `mask` once and only
  the subsets — don't reinvent it with a slower nested loop.
- **Interview cue:** "$n$ is at most 15–20 and you must track which items are used" is a flashing sign
  for bitmask DP. Name the state `dp[mask]` immediately.

## Practice

1. **Easy:** How many subsets does a 4-element set have? Confirm with the demo.
2. **Easy:** Write the mask for the subset $\{0, 2, 3\}$ of items 0–4, and give its decimal value.
3. **Medium:** Enumerate all submasks of `0b1011` using `sub = (sub - 1) & mask`.
4. **Medium:** Given `n` people and `k` seats per row, count valid seatings where no two chosen seats
   are adjacent (a per-row mask is "who is seated").
5. **Hard:** Solve "shortest path visiting all nodes" (LeetCode 847) with a BFS over states
   `(mask, node)`, finishing when `mask === (1 << n) - 1`.
