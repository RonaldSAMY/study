## The hook: 32 booleans for the price of one

A file's UNIX permissions — read, write, execute, for owner, group and others — are nine yes/no
answers. You *could* store nine booleans, but the operating system stores **one number**: `755`. Each
bit is a switch, the whole set fits in a register, and checking "can the owner write?" is a single AND.
This packing trick — a **bitmask** — is how permissions, feature toggles, game states, and graphics
colors are stored everywhere.

<Callout type="intuition" title="The one-sentence idea">
Give each yes/no setting its own **bit position**; then one integer holds the whole set, and OR / AND /
XOR with a mask grants, tests, or toggles any setting in one operation.
</Callout>

## Intuition: name the bits, then operate

Define each flag as a distinct power of two — `1 << 0`, `1 << 1`, `1 << 2`, … — so no two overlap. Now
the four moves from the bit-tricks lesson get *meaning*:

- **Grant** a flag: `value | FLAG` — OR turns its bit on.
- **Revoke** a flag: `value & ~FLAG` — AND with the complement turns it off.
- **Toggle** a flag: `value ^ FLAG` — XOR flips it.
- **Test** a flag: `(value & FLAG) !== 0` — AND keeps it only if present.

Because each flag is a separate bit, you can combine masks: `READ | WRITE` is a single value meaning
"both," and `(value & (READ | WRITE)) === (READ | WRITE)` asks "are *all* of these on?"

## How it works

Start with `READ + EXEC = 0b00000101`. Grant `WRITE` (`1 << 1 = 0b00000010`) with OR:

```
  00000101   (READ + EXEC)
| 00000010   (WRITE mask)
  --------
  00000111   (READ + WRITE + EXEC)
```

Now revoke `EXEC` (`1 << 2`) by ANDing with its complement `~(1 << 2) = 0b11111011`:

```
  00000111
& 11111011   (~EXEC)
  --------
  00000011   (READ + WRITE)
```

Every other bit survived untouched — that is the whole appeal of masks.

## The real implementation

The course file wraps these moves in tiny helpers — the exact functions a permissions system would
export:

```ts
// /dsa/16-bit-manipulation/bit-flags-masks.ts
function hasFlag(value: number, flag: number): boolean {
    return (value & flag) !== 0;
}
function setFlag(value: number, flag: number): number {
    return value | flag;
}
function clearFlag(value: number, flag: number): number {
    return value & ~flag;
}
function toggleFlag(value: number, flag: number): number {
    return value ^ flag;
}
// "are ALL of these flags set?"  vs  "is ANY set?"
function hasAllFlags(value: number, flags: number): boolean {
    return (value & flags) === flags;
}
function hasAnyFlag(value: number, flags: number): boolean {
    return (value & flags) !== 0;
}
```

Run the full file: `npx tsx 16-bit-manipulation/bit-flags-masks.ts`.

## Complexity

Every flag operation is a single masked instruction, no matter how many flags the integer holds:

$$
T(n) = O(1), \qquad S = O(1) \ \text{for up to } w \text{ flags in one } w\text{-bit word.}
$$

| Operation | Expression | Time |
|-----------|------------|------|
| Grant | `v \| FLAG` | $O(1)$ |
| Revoke | `v & ~FLAG` | $O(1)$ |
| Toggle | `v ^ FLAG` | $O(1)$ |
| Test one / any | `(v & FLAG) !== 0` | $O(1)$ |
| Test all | `(v & MASK) === MASK` | $O(1)$ |

Compared with an array or set of booleans, a bitmask is both smaller (one word vs. many) and faster
(no iteration, no hashing).

## Common pitfalls & interview tips

- **Flags must be distinct powers of two.** If two share a bit, setting one secretly sets the other.
  Define them as `1 << 0`, `1 << 1`, … and never as `3` or `5`.
- **"Has any" vs "has all" are different.** `(v & mask) !== 0` means *at least one*; `(v & mask) === mask`
  means *every one*. Interviewers love this distinction.
- **Packing fields needs shift *and* mask.** To read a sub-field, shift it down to the ones place
  *then* AND off the higher bits: `(packed >> 8) & 0xff`. Forgetting the mask leaks neighbouring fields.
- **Watch the 32-bit, signed ceiling.** Flag 31 sets the sign bit; beyond 32 flags you need `BigInt` or
  multiple words.

## Practice

1. **Easy:** With `READ=1, WRITE=2, EXEC=4`, what integer represents read + execute?
2. **Easy:** Given `perms = 6`, does it include `WRITE`? Include `READ`?
3. **Medium:** Write `hasAll(value, mask)` and use it to check a user has *both* `ADMIN` and `AUDIT`.
4. **Medium:** Pack an `(x, y)` coordinate (each 0–255) into one integer and write the extractors.
5. **Hard:** Implement a `FilePermissions` class that converts between an octal number like `0o755` and
   a symbolic string like `rwxr-xr-x` using shifts and masks.
