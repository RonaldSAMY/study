## The hook: tidying a tangled calendar

Open any shared calendar and you will find overlapping blocks — a meeting bleeding into a call bleeding
into a "do not disturb." To answer "when am I actually busy?" you want to fuse those overlapping blocks
into clean, non-overlapping stretches. The trick is almost insultingly simple: **sort the blocks by when
they start, then sweep once**, growing the current block whenever the next one touches it and starting a
fresh block when there is a gap. One pass, no backtracking.

<Callout type="intuition" title="The one-sentence idea">
Sort intervals by start time and sweep left to right keeping one open block — extend it on overlap, emit
it and open a new one on a gap — and you merge the whole set in a single linear pass.
</Callout>

## Intuition: the sort is the algorithm

Activity selection sorted by *finish* time to maximize a count. Merging is a different goal, so it uses a
different key: **start time**. Why is one left-to-right pass enough?

Once the intervals are sorted by start, any interval that overlaps the current open block must start at or
before that block's end. And crucially, no *later* interval can reach back to overlap a block we already
closed, because later intervals start even further right. That monotonicity is the greedy-choice
property in disguise: the decision to close a block is permanent and safe. The exchange argument is
trivial here — there is only one correct merged set, and the sorted sweep produces it.

The same "sort by an endpoint, sweep once" template solves a whole family:

- **Merge** overlapping intervals — sort by start, extend or open.
- **Maximum non-overlapping count** — sort by *end* (that is activity selection).
- **Minimum rooms / peak concurrency** — process start and end *events* with a running counter.

## How it works

Intervals `[1,3] [2,6] [8,10] [15,18] [9,12]`, sorted by start become `[1,3] [2,6] [8,10] [9,12] [15,18]`:

```
open [1,3]
[2,6]: 2 ≤ 3 → overlap, extend end to 6  → open [1,6]
[8,10]: 8 > 6 → gap, emit [1,6], open [8,10]
[9,12]: 9 ≤ 10 → overlap, extend end to 12 → open [8,12]
[15,18]: 15 > 12 → gap, emit [8,12], open [15,18]
result: [1,6], [8,12], [15,18]
```

## The real implementation

From the course file — sort by start, extend the last block on overlap, push a new one otherwise:

```ts
// /dsa/14-greedy/interval-scheduling.ts
function mergeIntervals(intervals: Interval[]): Interval[] {
    if (intervals.length <= 1) return intervals;

    // Sort by start time
    intervals.sort((a, b) => a[0] - b[0]);

    const result: Interval[] = [intervals[0]];

    for (let i = 1; i < intervals.length; i++) {
        const last = result[result.length - 1];
        const current = intervals[i];

        if (current[0] <= last[1]) {
            last[1] = Math.max(last[1], current[1]);  // overlapping → extend the block
        } else {
            result.push(current);                     // gap → start a new block
        }
    }
    return result;
}
```

Run the full file: `npx tsx 14-greedy/interval-scheduling.ts`.

## Complexity

Sort, then a single sweep that touches each interval once:

$$
T(n) = O(n \log n), \qquad S(n) = O(n).
$$

| Task | Sort by | Pass | Time |
|------|---------|------|------|
| Merge overlapping | start | extend or open | $O(n \log n)$ |
| Max non-overlapping | end | keep if compatible | $O(n \log n)$ |
| Minimum rooms | events ($+1$ / $-1$) | running max | $O(n \log n)$ |

The $O(n)$ space is just the output list; the sweep itself needs only the last open block.

## Common pitfalls & interview tips

- **Wrong sort key for the goal.** Merging needs *start*; maximizing a count needs *end*; minimum rooms
  needs *events*. Naming the goal first tells you the key.
- **Touching endpoints.** Decide whether `[1,4]` and `[4,5]` merge. Using `<=` fuses them; `<` keeps them
  apart. State your convention out loud.
- **Mutating shared arrays.** When you extend `last[1]`, make sure `last` is a copy, not a reference into
  the caller's data — otherwise you silently corrupt the input.
- **Forgetting to sort.** The sweep is only correct on sorted input; an unsorted pass misses overlaps that
  appear out of order.
- **Interview cue:** "merge / insert into / intersect / minimum rooms for intervals" is the tell. Reach for
  sort-then-sweep and decide the key from the objective.
